# M013 Critical/High Security Evidence Matrix

## Scope

M013 covers the critical/high findings F-01 through F-11 for WhatsApp management authentication, WuzAPI media SSRF prevention, cross-doctor/cross-patient authorization, public quiz session integrity, private upload/report serving, generated report artifact privacy, and report ownership closure.

This matrix is derived from the M013 context, roadmap, requirements register, completed S01-S05 summaries, S06/T01 closeout, and the reusable security boundary helper. It is the reviewer-facing inspection surface for R001 through R011. R012, R013, and R014 remain deferred follow-ups; R015, R016, R017, and R018 are non-goals/out-of-scope for this milestone unless a later requirement explicitly reopens them.

The document intentionally uses synthetic labels, file paths, command names, requirement IDs, and finding IDs only. It must not copy PHI, message bodies, quiz answers, free-text flow response values, raw tokens, cookies, private filesystem paths, sensitive URLs, or secrets from tests or logs.

## R010 Reusable Negative Isolation Proof

R010 is the milestone-wide reusable negative-proof requirement. The central helper is `backend-hormonia/tests/api/v2/security_boundary_helpers.py`, which creates a two-doctor/two-patient/admin fixture, one message per patient, auth override helpers, and response-leak assertions.

Suites that consume this helper directly include:

- `backend-hormonia/tests/api/v2/test_patient_ownership_boundary.py` for messages, flow responses, and flow overrides.
- `backend-hormonia/tests/api/v2/test_quiz_link_session_boundary.py` for authenticated quiz ownership and public quiz state boundaries.
- `backend-hormonia/tests/api/v2/test_monthly_quiz_compatibility.py` for compatibility state coverage built from the same fixture pattern.

Additional two-doctor/two-patient or owner-vs-foreign negative suites use the same boundary pattern even when their setup is local to the file: `backend-hormonia/tests/api/v2/test_report_ownership_closure.py`, `backend-hormonia/tests/api/v2/test_private_upload_serving.py`, and the report/enhanced-report compatibility suites listed in F-07 through F-09. Together they prove unauthorized/foreign access fails while assigned-doctor, owner, admin, or legitimate public quiz flows still pass.

## R011 PHI-Safe Diagnostics Contract

Denied or failed auth, authorization, SSRF, private-file, report, and quiz paths may log or return only investigation-safe fields: stable non-PHI IDs, actor role or actor ID when safe, resource ID, task name, HTTP status, task/report status, generic reason, and failure type.

Denied or failed paths must not expose PHI, patient names, phone numbers, message content, quiz answers, free-text flow response values, raw or derived tokens, cookies, signed state values, private filesystem paths, sensitive URLs, secrets, free-form report labels, or report bodies. This applies equally to HTTP responses, structured log extras, task result fields, redirects, export status payloads, and pytest failure diagnostics.

The matrix rows point to the primary negative tests for each boundary. The executable documentation validation below also guards this artifact against unsafe sentinel strings and missing coverage IDs.

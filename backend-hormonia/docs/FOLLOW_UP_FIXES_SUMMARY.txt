┌──────────────────────────────────────────────────────────────────────────────┐
│ 💡 NOTAS TÉCNICAS IMPORTANTES                                               │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  async_to_sync vs asyncio.run:                                              │
│                                                                              │
│  ❌ asyncio.run():                                                           │
│     - Cria novo event loop sempre                                           │
│     - Conflita com event loops existentes                                   │
│     - RuntimeError em workers Celery                                        │
│     - Não é thread-safe                                                     │
│                                                                              │
│  ✅ async_to_sync():                                                         │
│     - Usa event loop do worker se existir                                   │
│     - Cria loop temporário apenas se necessário                             │
│     - Thread-safe para workers Celery                                       │
│     - Melhor integração com frameworks assíncronos                          │
│     - Performance otimizada                                                  │
│                                                                              │
│  Estratégia de Persistência:                                                │
│                                                                              │
│  PRIMARY:   Redis (distributed, persistent)                                 │
│  FALLBACK:  In-Memory (local, volatile)                                     │
│  SYNC:      Bidirectional (rehydrate + sync)                                │
│                                                                              │
│  Vantagens:                                                                  │
│  ✓ Alta disponibilidade (funciona sem Redis)                               │
│  ✓ Zero data loss (sync automático)                                        │
│  ✓ Graceful degradation (fallback transparente)                            │
│  ✓ Auto-recovery (rehydrate + sync)                                        │
│                                                                              │

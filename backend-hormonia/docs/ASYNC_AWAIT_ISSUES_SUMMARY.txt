ISSUE #5: time.sleep() in Dead Letter Queue Handler
Location: app/resilience/retry/dead_letter.py:193
Severity: HIGH - Thread leak, ungraceful shutdown
Impact: Message loss on shutdown, daemon thread blocking

Pattern Issue:
  def _requeue_message(self, message):
      def requeue_after_delay():
          time.sleep(self.retry_backoff)  # LINE 193
          self._queue.put_nowait(message.id)
      thread = threading.Thread(target=requeue_after_delay, daemon=True)
      ^ WRONG: Daemon thread with blocking sleep

Correct Pattern:
  def _requeue_message(self, message):
      timer = threading.Timer(
          self.retry_backoff,
          self._do_requeue,
          args=[message]
      )
      timer.daemon = False  # Allow graceful shutdown
      self._pending_timers.append(timer)
      timer.start()

ISSUE #9: Poor Error Context Detection
Location: app/celery_app.py:423-425
Severity: MEDIUM - Confusing errors, Hidden failures
Impact: Hard to debug, misleading error messages

Pattern Issue:
  try:
      asyncio.get_running_loop()
      raise RuntimeError("Cannot call from async")
  except RuntimeError:
      asyncio.run(coro)  # Catches BOTH errors!
  ^ WRONG: Generic except catches the raise above

Correct Pattern:
  try:
      loop = asyncio.get_running_loop()
      raise RuntimeError("...")
  except RuntimeError as e:
      if "There is no current event loop" in str(e):
          return asyncio.run(coro)
      raise
  ^ RIGHT: Specific error checking

ISSUE #11: ThreadPoolExecutor Leak
Location: app/services/patient/crud_service.py:38-119
Severity: LOW - Resource leak on shutdown
Impact: Minor resource leak

Pattern Issue:
  _cache_executor = ThreadPoolExecutor(max_workers=2)
  _cache_executor.submit(_run_in_thread)  # Fire and forget
  ^ WRONG: Executor never shuts down

Correct Pattern:
  def __init__(self):
      self._executor = ThreadPoolExecutor(max_workers=2)
      self._pending = set()

  async def cleanup(self):
      for future in self._pending:
          await asyncio.wait_for(asyncio.wrap_future(future), timeout=5)
      self._executor.shutdown(wait=True)

Pattern Issue:

      loop = asyncio.get_running_loop()
      task = loop.create_task(coro)
  except RuntimeError:
      asyncio.run(coro)
      return None
  ^ CORRECT but confusing

Better Pattern:
  """Safe pattern for dual-context (sync + async)
  1. Try get_running_loop() - succeeds only if in async
  2. If raises RuntimeError, we're in sync context
  3. THEN asyncio.run() is safe
  """

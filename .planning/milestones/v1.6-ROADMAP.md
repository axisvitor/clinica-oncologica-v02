# Milestone v1.6: WuzAPI Migration

**Status:** ✅ SHIPPED 2026-03-03

## Overview

## Milestone Summary

**Key Decisions:**
- Hard cut to WuzAPI (no dual-provider mode or feature toggles).
- Standardize WuzAPI secret/config access through centralized Pydantic settings.
- Keep unsupported contacts sync route for compatibility, but return explicit HTTP 501.
- Preserve Evolution modules only as tombstones with deterministic ImportError guidance.

**Issues Resolved:**
- WuzAPI outbound migration completed across direct, queue, and idempotent flows.
- Evolution provider runtime fully removed and guarded by CI/source checks.
- Audit gap M-1 fixed (webhook secret source consistency).
- Audit gap M-2 fixed (contacts sync no longer returns misleading HTTP 200).

**Issues Deferred:**
- Live-provider operational verification remains manual: real send/media, real webhook HMAC, real QR pairing, and LID DLQ production observability.

**Technical Debt Incurred:**
- None material in code surface; remaining debt is environment validation that requires live WuzAPI infrastructure.

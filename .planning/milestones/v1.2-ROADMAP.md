## Milestone Summary

**Key Decisions:**
- google-adk DEFERRED to v1.3: irresolvable OTel cap conflict + Pydantic 2.11+ FastAPI schema failures + 300-400 MB GCP footprint
- pydantic-ai-slim[google,retries]>=1.63.0,<2.0.0: correct variant with pin <2.0.0 (V2 planned April 2026)
- PromptedOutput(SentimentResult) for SentimentAgent: Gemini cannot use tool-calling + native structured output simultaneously
- PIISafeAgent wrapper MANDATORY: only sanctioned way to call any agent; direct .run() calls blocked by CI lint rule
- Flow graphs replaced with direct async Python (not ADK): 10-15 lines each, identical runtime semantics
- Hard-cut GeminiClient from LangChain to google-genai without feature toggles
- Deleted AI_FRAMEWORK and AI_FLOW_FRAMEWORK without deprecation to enforce hard-cut migration

**Issues Resolved:**
- LangGraph import dependency coupling (AST audit + phased migration)
- Consensus dead code accumulation (full deletion)
- LangChain SDK overhead (direct google-genai replacement)
- Redis PHI checkpoint data (LGPD-compliant purge with audit logging)
- Celery event loop closure errors (run_sync bridge with loop repair)

**Issues Deferred:**
- Google ADK installation (waiting for issue #3615 resolution)
- Full AsyncSession migration (42+ remaining methods)
- 60+ files >500 lines splitting
- Physician availability hours model

**Technical Debt Incurred:**
- PromptedOutput validation confidence against gemini-2.5-flash is MEDIUM (needs spike test)
- 9 tombstoned LangGraph modules remain in codebase (ImportError sentinels)

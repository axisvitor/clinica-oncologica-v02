produces log entries with these keys: `step`, `action`, `status`, `timestamp`, and optionally
`message`. The current `SagaLogEntry` TypedDict has `operation` instead of `action` and `error`
instead of `message`.

Fix the TypedDict to match runtime:
```python
class SagaLogEntry(TypedDict):

    step: int
    action: str           # was "operation" — matches add_log_entry() parameter name
    status: str
    message: Optional[str]  # was "error" — matches add_log_entry() optional parameter
```

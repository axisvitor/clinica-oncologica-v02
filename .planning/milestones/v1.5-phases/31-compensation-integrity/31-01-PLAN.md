# Plan 31-01: Map Steps to Compensation Handlers and Verify Reverse-Order Rollback

## Goal

Produce a verification test module that statically confirms every active saga forward step has a matching compensation handler (COMP-01), and that partial failure at any step triggers compensation in correct reverse execution order (COMP-02). The tests serve as living documentation and regression guards.

## Must-Haves (goal-backward)

- [ ] A test proves the 1:1 mapping between active forward steps and compensation handlers — step 1 maps to `compensate_patient`, step 3 maps to `compensate_flow`, step 4 maps to `compensate_message`, and deprecated step 2 is explicitly documented as intentionally uncompensated
- [ ] A test proves compensation at `current_step=4` triggers handlers in order 4->3->1 (reverse of forward 1->3->4)
- [ ] Tests cover edge cases: `current_step=1` runs only `compensate_patient`; `current_step=3` runs `compensate_flow` + `compensate_patient`; `current_step=0` runs no handlers
- [ ] All tests pass: `cd backend-hormonia && python -m pytest tests/unit/orchestration/test_saga_compensation_integrity.py -x -q`

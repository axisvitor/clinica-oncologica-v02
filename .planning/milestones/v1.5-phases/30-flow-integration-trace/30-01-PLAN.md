**The complete chain to trace:**

```
POST /api/v2/patients/ (crud.py)
  -> get_onboarding_coordinator(db, saga_orchestrator) (onboarding_factory.py)
  -> OnboardingCoordinator.create_patient(patient_data, doctor_id, current_user, idempotency_key) (coordinator.py)
    -> integrity_service.validate_patient_data() -- Step 1
    -> saga_orchestrator.execute_patient_onboarding_saga() -- Step 2
      -> acquire_lock("saga:onboarding:{doctor}:{phone_hash}")
      -> Create PatientOnboardingSaga record
      -> step_executor.step_create_patient() (steps.py)
      -> step_executor.step_initialize_flow() (steps.py)
        -> flow_service.initialize_default_flow() (flow_service.py)
          -> _select_flow_type(patient) -> FlowType.ONBOARDING
          -> flow_engine.enroll_patient() (operations.py)
            -> Create PatientFlowState record
        -> flow_service.activate_patient() (flow_service.py)
      -> step_executor.step_send_welcome_message() (steps.py)
        -> message_service.schedule_message()
      -> Commit or Compensate on failure
```

```markdown
### Handoff N: [Caller] -> [Callee]

| Attribute | Value |

| Session | AsyncSession / Session / db: Any |
| Status | OK / FINDING: [description] |
```

- `db: Any` typing that obscures the actual session type

  <action>
    **1. Create `app/services/flow/_flow_functions.py`** — two async functions that replicate the graph's linear node execution without LangGraph runtime:

    - `async def run_flow_message(*, patient_id: UUID, day_number: int, flow_kind: str, handler: Any) -> Dict[str, Any]`: builds state dict `{"patient_id": patient_id, "day_number": day_number, "flow_kind": flow_kind, "result": None, "error": None}`, builds config dict `{"configurable": {"thread_id": f"flow_message:{patient_id}:{flow_kind}:{day_number}", "handler": handler}}`, calls `load_flow_context(state, config=config)`, merges returned dict into state, short-circuits if `state.get("result")` is a non-empty dict after load, then calls `dispatch_send_mode(state, config=config)`, merges result into state, and returns `state["result"]` (raises ValueError if not a dict).

    - `async def run_flow_response(*, patient_id: UUID, response_context: Optional[Dict[str, Any]], handler: Any) -> Dict[str, Any]`: builds state dict `{"patient_id": patient_id, "result": None, "error": None}` plus `"response_context": response_context` if not None, builds config dict `{"configurable": {"thread_id": f"flow_response:{patient_id}", "handler": handler}}`, calls `load_response_context(state, config=config)`, merges, short-circuits if result present, then calls `dispatch_response_continuation(state, config=config)`, merges, returns `state["result"]`.

    Both functions import the four node functions from `app.ai.langgraph.nodes` (this import still works in 12-01 because LangGraph packages are still installed). The config dict must contain both `thread_id` and `handler` keys inside `configurable` because `_require_handler(config)` expects both.

    **2. Add `AI_FLOW_FRAMEWORK` to `IntegrationsSettings`** in `app/config/settings/integrations.py`:
    ```python
    AI_FLOW_FRAMEWORK: str = Field(
        default="legacy",
        description=(
            "Flow orchestration framework: 'legacy' uses LangGraph StateGraph, "
            "'direct' uses direct async Python functions. "
            "Toggle at runtime via environment variable."
        ),
    )
    ```
    Place it right after the existing `AI_FRAMEWORK` field. Also add `AI_FLOW_FRAMEWORK=legacy  # 'legacy' or 'direct' - toggles flow orchestration` to `.env.example` right after the `AI_FRAMEWORK` line.

    **3. Wire feature-flagged call sites in `sequential_message_handler.py`:**

    Add a private method:
    ```python
    def _use_direct_flow_functions(self) -> bool:
        from app.config import settings
        return getattr(settings, "AI_FLOW_FRAMEWORK", "legacy") == "direct"
    ```

    In `send_day_messages()`, add the direct branch BEFORE the existing `try:` block:
    ```python
    if self._use_direct_flow_functions():
        try:
            from app.services.flow._flow_functions import run_flow_message
            return await run_flow_message(
                patient_id=patient_id,
                day_number=day_number,
                flow_kind=flow_kind,
                handler=self,
            )
        except Exception as e:
            logger.exception("Error sending day messages via direct flow function")
            return {"status": "error", "message": str(e)}
    ```
    Leave the legacy `graph.ainvoke()` branch fully intact.

    In `handle_response_and_continue()`, add the same pattern:
    ```python
    if self._use_direct_flow_functions():
        try:
            from app.services.flow._flow_functions import run_flow_response
            return await run_flow_response(
                patient_id=patient_id,
                response_context=response_context,
                handler=self,
            )
        except Exception as e:
            logger.exception("Error handling response continuation via direct flow function")
            return {"status": "error", "message": str(e)}
    ```

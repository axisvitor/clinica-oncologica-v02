```python
class SentimentResult(BaseModel):
    sentiment: str = "neutral"
    confidence: float = 0.5
    emotional_indicators: List[str] = Field(default_factory=list)
    medical_concerns: List[str] = Field(default_factory=list)

```

Add field validators:
- `sentiment`: normalize to lowercase, constrain to {"positive", "negative", "neutral", "concerning"}, default to "neutral" if invalid. Use `@field_validator("sentiment", mode="before")`.
- `confidence`: clamp to [0.0, 1.0], default 0.5 if None. Use `@field_validator("confidence", mode="before")`.
- `medical_concerns`: normalize to list of non-empty strings (handle bool/str/list/None inputs). Use `@field_validator("medical_concerns", mode="before")`.

These validators mirror the existing `SentimentAnalysisResult` validators but use pydantic v2 `@field_validator` syntax (not v1 `@validator`).

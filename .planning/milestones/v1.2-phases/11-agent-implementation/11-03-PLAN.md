2. `@_humanize_agent.output_validator` guardrails (per CONTEXT.md: per-agent, not shared base):
   - Strip/normalize: `text = (result or "").strip()`
   - Length bounds from MESSAGE_HUMANIZED profile: min=6, max=1800
     - `len(text) < 6` -> `raise ModelRetry("Output too short -- regenerate")`
     - `len(text) > 1800` -> `raise ModelRetry("Output too long -- regenerate")`
   - Banned patterns: `for pattern in _BANNED_PATTERNS: if re.search(pattern, text): raise ModelRetry("Banned pattern in output -- regenerate")`
   - Prompt leak markers: `for marker in _PROMPT_LEAK_MARKERS: if marker in text: raise ModelRetry("Prompt echo detected -- regenerate")`
   - Brazilian Portuguese ending punctuation: `if not re.search(r"[.!?…][\"')\\]]*$", text.rstrip()):`
     - Self-repair first: append "." and return repaired text
     - This matches existing `_ENDING_PUNCTUATION_PATTERN` behavior but with self-repair instead of hard fail
   - Return validated text

3. `class HumanizeAgent(PIISafeAgent)`:
   - `_agent = _humanize_agent`
   - Method: `async def humanize(self, template: str, patient_name: str, patient_context: dict, conversation_history: list, personalization_hints: list, ai_instructions: str | None, deps: AIDeps) -> str`

   - Build context and prompt matching the existing `GeminiDomainClient.humanize_flow_message()` logic:
     - `context = {**(patient_context or {}), "patient_name": patient_name}`
     - `recent_interactions = _coerce_recent_interactions(context.get("recent_interactions"), fallback_history=conversation_history or [])`
     - `template_with_name = _replace_patient_name(template, patient_name)`

   - Call `return await self._safe_run(prompt, deps, operation="humanize")`

**VariationAgent (variation_agent.py):**

This agent has a unique characteristic: the 88% word overlap check that may fall back to deterministic output.

1. Module-level agent singleton:
```python
_variation_agent = Agent(
    model=None,
    output_type=str,
    deps_type=AIDeps,
    retries=1,
    output_retries=1,
    model_settings=ModelSettings(timeout=30.0),
    defer_model_check=True,
)
```

2. `@_variation_agent.output_validator` guardrails:
   - Same text guardrails as humanize/empathy: strip, length bounds (MESSAGE_STANDARD profile: min=3, max=1600), banned patterns, prompt leak markers
   - Ending punctuation check with self-repair
   - NOTE: The 88% overlap check is NOT in the output_validator (it needs access to recent_questions which is not available in ctx). Instead, it's handled post-_safe_run in the vary() method. This avoids research pitfall 5 (infinite ModelRetry loop).

3. `class VariationAgent(PIISafeAgent)`:
   - `_agent = _variation_agent`
   - Method: `async def vary(self, base_question: str, previous_questions: list, patient_context: dict, ai_instructions: str | None, deps: AIDeps) -> str`
   - Inside: import from `app.ai.agents.helpers`:
     - `_coerce_recent_interactions`, `_extract_recent_questions`, `_is_too_similar_to_recent`, `_build_non_repetitive_question`, `_replace_patient_name`
   - Import `build_question_variation_prompt` from `app.ai.agents.helpers`
   - Build prompt matching existing `GeminiDomainClient.generate_varied_question()`:
     - `context = patient_context or {}`
     - `patient_name = context.get("patient_name") or context.get("name") or ""`
     - `recent_interactions = _coerce_recent_interactions(context.get("recent_interactions"), fallback_history=previous_questions or [])`
     - `recent_questions = _extract_recent_questions(recent_interactions, previous_questions or [])`
     - `question_with_name = _replace_patient_name(base_question, patient_name)`
     - `prompt = build_question_variation_prompt(base_question=question_with_name, ai_instructions=ai_instructions, recent_interactions=recent_interactions)`
   - Call `output = await self._safe_run(prompt, deps, operation="variation")`
   - **Post-call 88% overlap check (critical AGENT-03 requirement):**
     - `if _is_too_similar_to_recent(output, recent_questions):`
     - `    output = _build_non_repetitive_question(base_question, recent_questions)`
     - This matches the existing fallback behavior in `GeminiDomainClient.generate_varied_question()`
   - Return `output`

## Cross-Phase Integration

The actual Phase 44 -> 47 wiring looks coherent in repository evidence:

1. The canonical API route `/api/v2/adk/run` builds runtime/session/invocation context in `backend-hormonia/app/api/v2/routers/adk.py` and delegates to `PIISafeADKWrapper.safe_run(...)`.
2. `PIISafeADKWrapper` normalizes the context into an `ADKToolRunRequest` and calls `run_adk_tool(...)` in `backend-hormonia/app/ai/adk/wrapper.py`.
3. `run_adk_tool(...)` enforces unsupported-tool handling, session lifecycle, deterministic terminal statuses, and metrics emission in `backend-hormonia/app/ai/adk/runtime.py`.
4. `/metrics` exposes the default Prometheus registry in `backend-hormonia/app/api/v2/metrics.py`, which is where the ADK metrics module registers latency/counter/gauge series.
5. CI gates deployment on both `test-backend` and `smoke-adk`, and then fans into `build-backend` and `ci-status` in `.github/workflows/ci.yml`.

No repository-level cross-phase break was found in the wiring itself.

await self.connection_manager.broadcast_to_patient_room(room_id, payload)

**Fix 3 -- `_handle_user_message()` (lines ~330-341):**
Replace the ENTIRE method body. The current implementation manually iterates `self.connection_manager.connections` using dict `.get()` on `ConnectionInfo` dataclass objects (which don't have `.get()`), then calls non-existent `send_personal_message()`. Replace with a single call to `broadcast_to_user()` which handles the user_id -> connection lookup correctly:

Change:
```python
async def _handle_user_message(self, user_id: str, data: Dict[str, Any]):
    """..."""
    payload = data.get("payload", {})

    # Get all connections for this user
    user_connections = [
        conn_id
        for conn_id, conn_data in self.connection_manager.connections.items()
        if conn_data.get("user_id") == user_id
    ]

    # Send to each connection
    for conn_id in user_connections:
        await self.connection_manager.send_personal_message(payload, conn_id)
```
To:
```python
async def _handle_user_message(self, user_id: str, data: Dict[str, Any]):
    """
    Handle user-specific message - send to all user's connections.

    Args:
        user_id: User identifier
        data: Message data
    """
    payload = data.get("payload", {})
    await self.connection_manager.broadcast_to_user(user_id, payload)
```

CRITICAL CONSTRAINTS:
- Do NOT change the public API of `UnifiedWebSocketConnectionManager` -- only fix the callers.
- Do NOT remove the echo prevention in `_handle_pubsub_message()` (`if data.get("instance_id") == self.instance_id: return`) -- it is correct and prevents duplicate delivery.
- Do NOT change the publish methods (`publish_broadcast`, `publish_to_room`, `publish_to_user`) -- they are correct.
- Keep the docstrings and method signatures unchanged (only the bodies change).

  <action>
In `service_health.py`, replace the hardcoded `avg_task_duration_seconds=2.5` with a real Redis read.

1. Add import at top: `from app.core.redis_manager import redis_manager`
2. Create an async helper function `_read_avg_task_duration() -> float`:
   ```python
   async def _read_avg_task_duration() -> float:
       """Read rolling average task duration from Redis. Returns 0.0 on error or empty."""
       try:
           client = await redis_manager.get_async_client()
           samples = await client.lrange("celery:metrics:avg_task_duration", 0, -1)
           if not samples:
               return 0.0
           durations = [float(s) for s in samples]
           return round(sum(durations) / len(durations), 3)
       except Exception:
           return 0.0
   ```
3. In `check_worker_health()` (async function, line ~80), replace `avg_task_duration_seconds=2.5,  # TODO: Calculate actual average` with:
   ```python
   avg_task_duration_seconds=await _read_avg_task_duration(),
   ```
4. Remove the TODO comment since it's now implemented.

CRITICAL CONSTRAINTS:
- `check_worker_health` is already `async def`, so `await` is safe here.
- Use `redis_manager.get_async_client()` (async client) since we're in an async context.
- Return `0.0` on any exception (empty list, Redis down, parse error) -- health endpoint must not crash.
- The `round(..., 3)` prevents floating-point noise in JSON response.

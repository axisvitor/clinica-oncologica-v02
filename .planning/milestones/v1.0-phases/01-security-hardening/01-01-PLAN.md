from app.dependencies.auth_dependencies import (
    get_admin_user,          # for admin-only mutation endpoints

   - **No auth (keep as-is):** `/health` endpoint (Railway health probe), `/export/prometheus` (Prometheus scraping — OWASP standard is network-level protection, not app auth).

4. **Remove any now-unused imports** that the old `get_admin_user` needed (e.g., direct `User` model import if only used there, `get_db` if only used in the deleted function). Keep imports that are still used by other parts of the file.

5. **Do NOT change the endpoint route paths or response schemas** — only the auth dependency injection.

Note: The canonical `get_admin_user` chains through `get_current_user_from_session` which validates the session in Redis + checks admin role. The canonical `get_current_active_user` validates session + checks user is active (any role). Per user decision, both admin and doctor roles are permitted for monitoring reads.

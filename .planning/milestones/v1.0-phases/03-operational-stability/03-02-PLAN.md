<objective>
Implement the atomic Lua script for the sliding window rate limiter, replacing the non-atomic Redis pipeline that has a documented race condition between ZCARD and ZADD.

Purpose: The current pipeline-based implementation allows concurrent requests to all read count=0 before any increment is visible, permitting brief bursts above the configured limit. The Lua script executes the entire check-then-increment as a single atomic operation on the Redis server.

Output: Modified `rate_limit_core.py` with Lua script registered at init and invoked in the `check_rate_limit` method's increment path.
</objective>

local key = KEYS[1]
local window_start = tonumber(ARGV[1])
local current_time = tonumber(ARGV[2])
local limit = tonumber(ARGV[3])
local window = tonumber(ARGV[4])
redis.call('ZREMRANGEBYSCORE', key, 0, window_start)
local count = redis.call('ZCARD', key)
if count < limit then
    redis.call('ZADD', key, current_time, tostring(current_time))

end
return {0, count}
"""

The enhanced_analytics.py and performance.py use service factory patterns with `Depends(get_db)`. All must be migrated to close the global sync DI gap.
Output: All 7 files with zero `Depends(get_db)`, zero `db.query(`, using `await db.execute(select(...))` patterns.
</objective>

Standard async patterns (established in Phases 24-26):
```python
from app.core.database.async_engine import get_async_db
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select, func

db: AsyncSession = Depends(get_async_db)

# Read: select-based query
result = await db.execute(select(Model).where(Model.id == some_id))
item = result.scalar_one_or_none()

# Count
result = await db.execute(select(func.count(Model.id)).where(...))
total = result.scalar() or 0

# List
result = await db.execute(select(Model).where(...))
items = result.scalars().all()
```

CRITICAL PATTERNS:
1. `physicians/statistics.py` delegates to `PhysicianStatisticsService(db)` which uses `self.db.query(...)` extensively (8 calls). The service must be converted to use `await self.db.execute(select(...))` since it will receive AsyncSession.
2. `physicians/availability.py` delegates to `PhysicianAvailabilityService(db)` which uses `self.db.query(Appointment)` (3 calls). Same conversion needed.
3. `physicians/base.py` has `validate_physician_access()` utility that calls `db.query(User)` and `db.query(Patient)`. Convert to async select. Since this is called from async handlers, make it `async def` and `await` all DB calls.
4. `enhanced_analytics.py` has a DI factory `get_enhanced_analytics_service(db=Depends(get_db))` that creates `EnhancedAnalyticsService(db)`. The service itself is a separate module — if it uses `db.query()` internally, it will fail with AsyncSession. Inspect the service: if it only receives `db` and passes it through, changing DI is sufficient. If it calls `db.query()`, inline the DB operations in the router handlers instead.
5. `performance.py` has `get_performance_service(db=Depends(get_db))` factory creating `PerformanceService(db)`. Same considerations as enhanced_analytics.

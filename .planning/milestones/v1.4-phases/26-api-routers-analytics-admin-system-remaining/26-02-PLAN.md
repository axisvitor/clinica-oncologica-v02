<objective>
Migrate the two remaining analytics routers with special cases: dashboard.py (uses DashboardService) and reports.py (uses iter_db_dependency wrapper).

Purpose: `dashboard.py` has inline `db.query(Patient.id)` calls for doctor-patient scoping and a `get_dashboard_service` local DI factory that receives `db`. The service methods that run analytics do NOT use `db` (they receive `patient_ids` list and use pre-aggregated data), but the in-router `db.query(Patient.id)` calls must be converted to async. The `DashboardService` constructor accepts `db` but the specific methods called in the router (`get_patient_metrics`, `get_message_metrics`, etc.) are analytics helpers that call their own sync DB queries internally via the stored session — since DashboardService internals cannot be changed in this phase, the `patient_ids` queries are inlined async and the service is passed AsyncSession (its methods do NOT directly use `db.query` in the router handlers). `reports.py` uses `_get_db_dep()` wrapper around `iter_db_dependency(get_db)` — remove the wrapper entirely and use `Depends(get_async_db)` directly; convert `_check_patient_access` sync helper to async.
Output: Both files with zero `Depends(get_db)`, zero `db.query(`.
</objective>

async def _check_patient_access(
    db: AsyncSession, role: UserRole, user_id: UUID, patient_ids: List[UUID]

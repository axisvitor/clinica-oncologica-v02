<objective>
Migrate admin/users.py (11 handlers) to AsyncSession.

Purpose: `admin/users.py` is the largest admin router in Phase 26 with 11 handlers, all using `db=Depends(get_db)`, `UserRepository(db)`, and `db.query(User)`. `UserRepository` is a sync repository whose methods (`get`, `get_by_email`, `create`, `update`, `list`) call `db.query(...)` internally -- passing AsyncSession to it would cause MissingGreenlet. The strategy (established in Plan 26-04 for activity.py) is to inline async SQL: replace `user_repo.get(user_id)` with `select(User).where(User.id == user_id)`, replace `user_repo.get_by_email(email)` with `select(User).where(User.email == email)`, and so on. All write operations (`db.commit()`, `db.refresh()`, `db.rollback()`) must be awaited.
Output: `admin/users.py` with zero `Depends(get_db)`, zero `db.query(`, zero `UserRepository(`, all write ops awaited.
</objective>

Write operations (all must be awaited):
```python
# Create user:
user = User(**user_data.dict())
db.add(user)
await db.commit()
await db.refresh(user)

user.role = update_data.role

await db.refresh(user)

# Delete / deactivate user:

await db.commit()

# Or hard delete:

await db.commit()

await db.rollback()
```

Instead, inline all DB operations directly in the router handlers. `UserRepository.get()` maps to `select(User).where(User.id == id)`, `UserRepository.get_by_email()` maps to `select(User).where(User.email == email)`, etc. These are trivial 1:1 replacements.

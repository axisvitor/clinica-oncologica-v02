CRITICAL for compensation.py: The `list_compensation_failures` handler reads `dialect_name = getattr(getattr(db, "bind", None), "dialect", None)`. With AsyncSession, `db.bind` is None. The dialect check is used to choose between JSONB `astext` and SQLite `cast + like` — this is a runtime DB dialect check. For AsyncSession, `db.bind` is `None`; use the async engine dialect instead:
```python
from app.core.database.async_engine import async_engine
dialect_name = async_engine.dialect.name
```
Or simply hardcode to use the JSONB path (production uses PostgreSQL).

CRITICAL for stats.py: `_status_count(db, status_value)` in utils.py uses sync `db.query(func.count(...))`. In stats.py, these calls must be replaced with inline async counts. Do NOT pass AsyncSession to `_status_count`. Instead, inline:
```python
# Replace: scheduled = _status_count(db, AppointmentStatus.SCHEDULED.value)
result = await db.execute(
    select(func.count(Appointment.id)).where(Appointment.status == AppointmentStatus.SCHEDULED.value)
)
scheduled = result.scalar() or 0
```
Apply same pattern for confirmed, in_progress, completed, cancelled.

Additionally, add an async `_status_count_async` to utils.py so future callers can use it:
```python
async def _status_count_async(db: AsyncSession, status_value: str) -> int:
    from sqlalchemy import select, func
    from sqlalchemy.ext.asyncio import AsyncSession
    from app.models.appointment import Appointment
    result = await db.execute(
        select(func.count(Appointment.id)).where(Appointment.status == status_value)
    )
    return result.scalar() or 0
```

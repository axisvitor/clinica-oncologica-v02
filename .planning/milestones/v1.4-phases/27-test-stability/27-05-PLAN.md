```python

def admin_user(db_session: Session):
    # Creates 1 admin user (email=admin@test.com)
    ...

def regular_users(db_session: Session):
    # Creates 25 users (email=user{0..24}@test.com)
    ...
```

The test `test_user_list_returns_paginated_response` currently asserts:
```python
assert data["total"] == 26      # 25 regular + 1 admin (HARDCODED)
assert data["total_pages"] == 3  # ceil(26/10)          (HARDCODED)
```

The endpoint `GET /api/v2/admin/users` uses `select(func.count(User.id))` with
filters applied — it counts ALL users in the database, not just fixture-created ones.

Root cause: `db_session` fixture wraps tests in a connection transaction but
`db_session.commit()` in fixtures escapes the outer transaction in SQLAlchemy 2.x
with StaticPool, causing users from earlier test files to persist.

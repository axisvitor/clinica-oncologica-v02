  <action>
    Create `app/core/database/dual_session.py` containing the `DualSessionMixin` class.

    This mixin provides helper methods that internally branch on `isinstance(self.db, AsyncSession)` so that service business code can call `self._execute(stmt)` without caring about sync vs async.

    **Implementation:**

    ```python
    """Dual-mode session mixin for services shared between API (async) and Celery (sync).

    Services inherit DualSessionMixin and call helper methods like self._execute(stmt)
    instead of directly using db.query() or await db.execute(). The mixin branches
    internally based on the session type passed at construction time.

    Usage:
        class MyService(DualSessionMixin):
            def __init__(self, db: Session | AsyncSession):
                self.db = db

            async def get_items(self):
                result = await self._execute(select(Item))
                return result.scalars().all()

    When called from sync context (Celery), the service must NOT be awaited.
    When called from async context (API), the service methods are awaited normally.
    The calling code (factory function or task) is responsible for using the correct
    invocation pattern — the mixin only handles DB operation dispatch.
    """

    from __future__ import annotations

    import logging
    from typing import Any, TypeVar, overload

    from sqlalchemy import Result
    from sqlalchemy.orm import Session
    from sqlalchemy.ext.asyncio import AsyncSession

    logger = logging.getLogger(__name__)

    T = TypeVar("T")


    class DualSessionMixin:
        """Mixin providing sync/async DB operation helpers.

        Subclasses MUST set `self.db` to either a `Session` or `AsyncSession` instance.

        All helper methods return coroutines when self.db is AsyncSession,
        and plain values when self.db is Session. This means:
        - In async context: `result = await self._execute(stmt)`
        - In sync context: `result = self._execute(stmt)`
        """

        db: Session | AsyncSession

        @property
        def is_async(self) -> bool:
            """Check if the current session is async."""
            return isinstance(self.db, AsyncSession)

        def _execute(self, stmt, **kwargs) -> Any:
            """Execute a SQLAlchemy statement.

            Returns Result for sync, Coroutine[Result] for async.
            """
            if self.is_async:
                return self.db.execute(stmt, **kwargs)
            return self.db.execute(stmt, **kwargs)

        def _scalars(self, stmt, **kwargs) -> Any:
            """Execute and return scalars.

            Returns ScalarResult for sync, Coroutine[ScalarResult] for async.
            """
            if self.is_async:
                return self.db.scalars(stmt, **kwargs)
            return self.db.scalars(stmt, **kwargs)

        def _get(self, entity: type[T], ident: Any) -> Any:
            """Get entity by primary key.

            Returns entity|None for sync, Coroutine[entity|None] for async.
            """
            if self.is_async:
                return self.db.get(entity, ident)
            return self.db.get(entity, ident)

        def _commit(self) -> Any:
            """Commit the current transaction.

            Returns None for sync, Coroutine[None] for async.
            """
            if self.is_async:
                return self.db.commit()
            self.db.commit()
            return None

        def _flush(self) -> Any:
            """Flush pending changes.

            Returns None for sync, Coroutine[None] for async.
            """
            if self.is_async:
                return self.db.flush()
            self.db.flush()
            return None

        def _refresh(self, instance: Any, **kwargs) -> Any:
            """Refresh an instance from the database.

            Returns None for sync, Coroutine[None] for async.
            """
            if self.is_async:
                return self.db.refresh(instance, **kwargs)
            self.db.refresh(instance, **kwargs)
            return None

        def _add(self, instance: Any) -> None:
            """Add an instance to the session. Sync in both modes."""
            self.db.add(instance)

        def _delete(self, instance: Any) -> Any:
            """Delete an instance.

            Returns None for sync, Coroutine[None] for async.
            """
            if self.is_async:
                return self.db.delete(instance)
            self.db.delete(instance)
            return None
    ```

    **Key design notes:**
    - For async sessions, the helper methods return the coroutine (not awaited). The caller uses `await self._execute(stmt)`. For sync sessions, they return the result directly. Since `await` on a non-coroutine raises TypeError, the caller must use `await` only in async methods — which is exactly the intended pattern (API handlers are async, Celery code is sync).
    - `_add()` is always sync because `Session.add()` and `AsyncSession.add()` are both synchronous operations.
    - `_execute()` and `_scalars()` look identical for both branches because SQLAlchemy's `Session.execute()` returns `Result` synchronously while `AsyncSession.execute()` returns a coroutine that resolves to `Result`. The `is_async` check is still useful for documentation and potential future branching.

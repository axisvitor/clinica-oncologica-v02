    **1. Create `backend-hormonia/app/dependencies/patient_services.py`:**

    ```python
    """Async service factory functions for patient-domain services.

    These factories use Depends(get_async_db) to inject AsyncSession into services.
    Each factory is a Depends()-compatible async function with flat injection only.

    Pattern:
        async def get_async_X_service(
            db: AsyncSession = Depends(get_async_db),
        ) -> XService:
            return XService(db)

    Sync factories remain in service_dependencies.py for Celery compatibility.
    New patient-domain async factories are added here as services are migrated in phases 22-27.

    Phase 21 establishes the pattern with the data integrity example.
    """

    from fastapi import Depends
    from sqlalchemy.ext.asyncio import AsyncSession

    from app.core.database import get_async_db


    async def get_async_data_integrity_service(
        db: AsyncSession = Depends(get_async_db),
    ):
        """Get DataIntegrityMonitoringService with AsyncSession.

        Used by API endpoints that need async data integrity checks.
        The service must inherit DualSessionMixin (migrated in Phase 22).
        """
        from app.services.data_integrity_monitoring import DataIntegrityMonitoringService

        return DataIntegrityMonitoringService(db)
    ```

    **2. Create `backend-hormonia/app/dependencies/flow_services.py`:**

    ```python
    """Async service factory functions for flow-domain services.

    These factories use Depends(get_async_db) to inject AsyncSession into services.
    Each factory is a Depends()-compatible async function with flat injection only.

    Pattern:
        async def get_async_X_service(
            db: AsyncSession = Depends(get_async_db),
        ) -> XService:
            return XService(db)

    Sync factories remain in service_dependencies.py for Celery compatibility.
    New flow-domain async factories are added here as services are migrated in phases 22-27.

    Phase 21 establishes the pattern with flow alerts and flow analytics examples.
    """

    from fastapi import Depends
    from sqlalchemy.ext.asyncio import AsyncSession

    from app.core.database import get_async_db


    async def get_async_flow_alerts_service(
        db: AsyncSession = Depends(get_async_db),
    ):
        """Get FlowAlertsService with AsyncSession.

        Used by API endpoints that need async flow alert operations.
        The service must inherit DualSessionMixin (migrated in Phase 22).
        """
        from app.services.flow_alerts import FlowAlertsService

        return FlowAlertsService(db)


    async def get_async_flow_analytics_service(
        db: AsyncSession = Depends(get_async_db),
    ):
        """Get FlowAnalyticsService with AsyncSession.

        Used by flow analytics API endpoints.
        The service must support AsyncSession (migrated in Phase 23).
        """
        from app.services.analytics import FlowAnalyticsService

        return FlowAnalyticsService(db)
    ```

    **Key design notes:**
    - All imports of services are lazy (inside function body) to avoid circular imports — this follows the same pattern used in `service_dependencies.py`.
    - Each factory takes only `db: AsyncSession = Depends(get_async_db)` — flat injection, no nested Depends chains (per CONTEXT decision).
    - The services are instantiated with the AsyncSession. They won't fully work in async mode until they inherit DualSessionMixin (Phase 22-23), but the factory pattern is established now.
    - Import `get_async_db` from `app.core.database` (canonical location, not `app.database`).
    - One file per domain per the locked CONTEXT decision.

<objective>
Fix the `alerts.type` column mapping mismatch that causes `UndefinedColumn` errors, unblocking the entire test suite.

The root cause: The Alert model declares `alert_type = Column("type", String(100))` which tells SQLAlchemy to generate SQL using `alerts.type` as the column name. However, the actual PostgreSQL production database has the column named `alert_type` (not `type`). This mismatch causes `psycopg.errors.UndefinedColumn: column "type" of relation "alerts" does not exist` when any test tries to insert or query alerts.

The fix: Remove the column name override so `alert_type` maps to the actual DB column `alert_type`. Verify the `description`/`message` mapping is correct. Add a conftest schema guard for PostgreSQL environments (following the established `_ensure_notifications_type_column` pattern) to ensure column alignment before tests run.

Output: Alert model with correct column mappings. All alert tests pass without `UndefinedColumn` errors.
</objective>

    **Implementation pattern** (follow the existing `_ensure_notifications_type_column` pattern):
    ```python
    def _ensure_alerts_columns(engine):
        """Ensure alerts table column names match the Alert model expectations.

        The Alert model was fixed to use `alert_type` (matching the actual DB column)
        instead of the previous incorrect mapping to `type`. This guard handles
        PostgreSQL test databases that may have stale column names.
        """
        if engine.dialect.name != "postgresql":
            return

        inspector = sa_inspect(engine)
        if not inspector.has_table("alerts"):
            return

        alert_columns = {col["name"] for col in inspector.get_columns("alerts")}

        # Fix: if DB has 'type' but not 'alert_type', rename it
        if "type" in alert_columns and "alert_type" not in alert_columns:
            print("[tests.conftest] Applying schema patch: rename alerts.type -> alerts.alert_type")
            with engine.begin() as connection:
                connection.execute(
                    text('ALTER TABLE alerts RENAME COLUMN "type" TO alert_type')
                )
        elif "type" not in alert_columns and "alert_type" not in alert_columns:
            print("[tests.conftest] Applying schema patch: add alerts.alert_type")
            with engine.begin() as connection:
                connection.execute(
                    text(
                        "ALTER TABLE alerts "
                        "ADD COLUMN IF NOT EXISTS alert_type VARCHAR(100)"
                    )
                )
                connection.execute(
                    text(
                        "UPDATE alerts SET alert_type = 'unknown' WHERE alert_type IS NULL"
                    )
                )
                connection.execute(
                    text(
                        "ALTER TABLE alerts ALTER COLUMN alert_type SET NOT NULL"
                    )
                )
    ```

    CRITICAL: The `"type"` in the `ALTER TABLE ... RENAME COLUMN "type"` MUST be quoted because `type` is a PostgreSQL reserved word. Use double quotes around it in the SQL string.

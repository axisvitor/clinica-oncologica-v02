<objective>
Add four missing awaitable wrapper methods (`delete`, `add`, `scalars`, `get`) to the
`SyncToAsyncSessionAdapter` class in both `tests/conftest.py` and `tests/api/critical/conftest.py`.

Currently, the adapter's `__getattr__` fallback returns the underlying sync method directly.
When a router calls `await db.delete(obj)`, the sync `Session.delete()` returns `None`, and
`await None` raises `TypeError: object NoneType can't be used in 'await' expression`.
Five routers use `await db.delete()`: alerts.py, appointments.py, flow_templates.py,
quiz_sessions.py, and quiz_templates.py. All five will fail through the test adapter.

The fix follows the exact pattern already used by the existing `commit()`, `flush()`,
`refresh()`, `rollback()`, `close()`, and `begin_nested()` wrappers in the same class.

Purpose: Close TEST-01 gap — test fixtures fully support AsyncSession endpoints.
Output: Both adapter classes updated with four new methods; regression test added.
</objective>

class SyncToAsyncSessionAdapter:
    """Async-compatible adapter over sync SQLAlchemy Session for tests."""

    def __init__(self, sync_session: Session):
        self._sync_session = sync_session

    @staticmethod
    def _awaitable(value=None):
        class _AwaitableValue:
            def __init__(self, resolved):
                self._resolved = resolved
            def __await__(self):
                async def _resolve():
                    return self._resolved
                return _resolve().__await__()

```python
    def delete(self, instance):
        self._sync_session.delete(instance)
        return self._awaitable()

    def add(self, instance):
        self._sync_session.add(instance)
        return self._awaitable()

    def scalars(self, statement, *args, **kwargs):
        return self._awaitable(self._sync_session.scalars(statement, *args, **kwargs))

    def get(self, entity, ident, **kwargs):
        return self._awaitable(self._sync_session.get(entity, ident, **kwargs))
```

Pattern explanation:
- `delete()` and `add()` call the sync method (which returns None) then return `self._awaitable()` with no value (defaults to None). This makes `await db.delete(obj)` resolve to None without TypeError.
- `scalars()` calls the sync `session.scalars()` (available in SQLAlchemy 2.x), wraps the ScalarResult in `self._awaitable()` so `result = await db.scalars(select(...))` gets a ScalarResult.
- `get()` calls the sync `session.get()` and wraps the result (instance or None) in `self._awaitable()`.

All four follow the identical pattern as the existing `commit()`, `flush()`, `refresh()`, `rollback()`, and `close()` methods already in the class.

# Milestone v1.4: AsyncSession & Test Stability

**Status:** SHIPPED 2026-02-28
**Phases:** 20-28
**Total Plans:** 54

## Overview

Complete the AsyncSession migration across all API-reachable code paths and fix the alerts schema to unblock the full test suite. Every API router now uses `get_async_db` (AsyncSession), all shared services support dual-mode Session/AsyncSession via DI, and the test fixtures provide a SyncToAsyncSessionAdapter with full awaitable contract support.

## Milestone Summary

**Decimal Phases:**
- Phase 28: Async Session Gap Closure (inserted after Phase 27 for audit gap closure)

**Key Decisions:**
- Celery tasks remain on sync Session (worker best practice — do not convert)
- Dual-mode DI pattern must land before service/router migrations
- Inlined async SQL in routers where sync repos couldn't accept AsyncSession

- Awaitable-resolution helpers to preserve Session and AsyncSession compatibility

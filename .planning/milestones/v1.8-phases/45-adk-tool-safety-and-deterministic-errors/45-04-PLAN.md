<objective>
Close the CRITICAL runner-path policy bypass gap identified in Phase 45 verification.

Purpose: The runner callback context merge in `runtime.py:_resolve_callback_context` and the tool dispatch context merge in `tools.py:_merge_context` both allow model-generated tool-call arguments (`context_json`, `context`) to overwrite operator-supplied policy fields (`tool_policy`, `policy`, `required_context_keys`). This defeats ADK-11 because a model-fabricated `context_json={"tool_policy": {}}` can erase the block policy before `before_tool_callback` evaluates it, and a fabricated `required_context_keys` satisfaction can bypass missing-context blocks.

Output: Hardened context merge functions that preserve operator-supplied policy keys as immutable, plus negative regression tests proving the bypass is closed.
</objective>

    **Step 2 — Fix `_resolve_callback_context` in `runtime.py`.**

    After merging `context_json` and `context` from `tool_args`, restore the original operator-supplied values for all protected keys. The fix pattern:

    ```python
    def _resolve_callback_context(
        *,
        request_context: dict[str, Any] | None,
        tool_args: dict[str, Any],
    ) -> dict[str, Any] | None:
        original = dict(request_context or {})
        merged_context = dict(original)

        context_json = tool_args.get("context_json")
        if isinstance(context_json, str):
            merged_context.update(_parse_callback_context_json(context_json))

        raw_context = tool_args.get("context")
        if isinstance(raw_context, dict):
            merged_context.update(raw_context)

        # Restore operator-supplied policy keys — tool-call args must not override them
        for key in _PROTECTED_POLICY_KEYS:
            if key in original:
                merged_context[key] = original[key]
            elif key in merged_context:
                del merged_context[key]

        return merged_context
    ```

    The critical logic: if the operator set a key, its value is restored after merge. If the operator did NOT set a key but the tool-call injected one, it is removed. This prevents both overwrite and fabrication.

    **Step 3 — Fix `_merge_context` in `tools.py`.**

    Add the same `_PROTECTED_POLICY_KEYS` constant in `tools.py` and apply the same restoration pattern:

    ```python
    _PROTECTED_POLICY_KEYS = frozenset({"tool_policy", "policy", "required_context_keys"})

    def _merge_context(runtime_context: dict[str, Any], context_json: str) -> dict[str, Any]:
        merged: dict[str, Any] = {}
        base_context = runtime_context.get("context")
        if isinstance(base_context, dict):
            merged.update(base_context)

        # Snapshot operator-supplied policy values before applying tool-call context
        operator_policy = {
            key: base_context[key]
            for key in _PROTECTED_POLICY_KEYS
            if isinstance(base_context, dict) and key in base_context
        }

        merged.update(_parse_context_json(context_json))

        # Restore operator-supplied policy keys — model context_json must not override them
        for key, value in operator_policy.items():
            merged[key] = value
        # Remove any fabricated policy keys not in the original operator context
        for key in _PROTECTED_POLICY_KEYS:
            if key not in operator_policy and key in merged:
                del merged[key]

        return merged
    ```

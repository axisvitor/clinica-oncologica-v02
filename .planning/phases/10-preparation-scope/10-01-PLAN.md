  <name>Task 1: Add LANGGRAPH_AUDIT env-gated import scanner to app/main.py</name>
  <files>backend-hormonia/app/main.py</files>
  <action>
Add a temporary LANGGRAPH_AUDIT env-gated import scanner at the TOP of `app/main.py` (before the existing imports, after any `from __future__` statement). The scanner:

1. Checks `os.getenv("LANGGRAPH_AUDIT") == "1"` — if not set, does nothing (zero runtime cost).
2. Uses `ast.parse` + `ast.walk` to scan all `app/**/*.py` files for imports matching these patterns:
   - Module names containing: `langgraph`, `langchain_core`, `langchain_google_genai`
   - ImportFrom nodes where `node.module` contains any of those patterns
   - Import nodes where any `alias.name` contains any of those patterns
3. Prints a sorted, deduplicated list to stdout: `LANGGRAPH AUDIT: [file1, file2, ...]`

The expected output (from research, verified against codebase):
- `app/ai/client.py` — langchain_core.messages, langchain_google_genai
- `app/ai/langgraph/consensus.py` — langgraph.graph
- `app/ai/langgraph/graphs.py` — langgraph.graph
- `app/ai/langgraph/nodes.py` — langchain_core.runnables
- `app/ai/langgraph/runtime.py` — langgraph.checkpoint
- `app/agents/communication/message_composer/composer.py` — app.ai.langgraph (indirect)
- `app/agents/patient/flow_coordinator/consensus_manager.py` — app.ai.langgraph (indirect)
- `app/ai/client_domain.py` — app.ai.langgraph (indirect)
- `app/core/lifespan.py` — app.ai.langgraph (indirect)
- `app/orchestration/consensus.py` — app.ai.langgraph (indirect)
- `app/services/ai/patient_summary_service.py` — langchain_core.messages, langchain_google_genai
- `app/services/analytics/data_extraction/service.py` — app.ai.langgraph (indirect)
- `app/services/enhanced_flow_engine.py` — app.ai.langgraph (indirect)
- `app/services/flow/sequential_message_handler.py` — app.ai.langgraph (indirect)
- `app/services/follow_up_system/generators/empathy.py` — app.ai.langgraph (indirect)
- `app/services/follow_up_system/generators/response.py` — app.ai.langgraph (indirect)

IMPORTANT: The scanner must detect both direct imports (`from langgraph.graph import ...`) AND indirect imports (`from app.ai.langgraph.graphs import ...`). Match `app.ai.langgraph` as a pattern too.

Use this exact code pattern (from research):
```python
import os
if os.getenv("LANGGRAPH_AUDIT") == "1":
    import ast, pathlib
    _AUDIT_PATTERNS = {"langgraph", "langchain_core", "langchain_google_genai"}
    _hits = []
    for _p in pathlib.Path("app").rglob("*.py"):
        try:
            _tree = ast.parse(_p.read_bytes())
        except SyntaxError:
            continue
        for _node in ast.walk(_tree):
            if isinstance(_node, (ast.Import, ast.ImportFrom)):
                _mod = getattr(_node, "module", None) or ""
                _names = [alias.name for alias in getattr(_node, "names", [])]
                if any(_pat in (_mod + " " + " ".join(_names)) for _pat in _AUDIT_PATTERNS):
                    _hits.append(str(_p))
                    break
    print("LANGGRAPH AUDIT:", sorted(set(_hits)))
```

Place this BEFORE the existing module docstring or imports in main.py. Use underscore-prefixed variables (_AUDIT_PATTERNS, _hits, _p, _tree, _node, _mod, _names) to avoid polluting the module namespace.

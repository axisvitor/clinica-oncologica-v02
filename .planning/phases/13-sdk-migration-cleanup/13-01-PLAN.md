  <name>Task 1: Migrate GeminiClient to google-genai SDK</name>
  <files>backend-hormonia/app/ai/client.py</files>
  <action>
    Replace the LangChain-based GeminiClient internals with google-genai SDK calls. This is a hard cutover per user decision — no toggle, no fallback.

    **_initialize_model() changes:**
    1. Remove `from langchain_google_genai import ChatGoogleGenerativeAI` (both top-level and in-method imports)
    2. Remove `from langchain_core.messages import HumanMessage` top-level import
    3. Add `from google import genai` and `from google.genai import types` imports
    4. Replace `ChatGoogleGenerativeAI(model=..., google_api_key=..., temperature=..., ...)` with:
       - `self._genai_client = genai.Client(api_key=self.api_key)`
       - `self._genai_config = types.GenerateContentConfig(temperature=settings.AI_GEMINI_TEMPERATURE, max_output_tokens=settings.AI_GEMINI_MAX_OUTPUT_TOKENS, top_p=settings.AI_GEMINI_TOP_P, top_k=settings.AI_GEMINI_TOP_K)`
       - `self.model = self._genai_client` (kept as sentinel for existing health check `if self.model is None`)
    5. Remove `_model_loop_id` tracking and `_ensure_model_for_loop()` logic entirely — google-genai Client is loop-agnostic (no need to reinitialize on loop change)

    **_generate_content_internal() changes:**
    1. Remove the `HumanMessage` wrapping: `messages = [HumanMessage(content=prompt)]`
    2. Replace `self.model.ainvoke(messages)` with `self._genai_client.aio.models.generate_content(model=self.model_name, contents=prompt, config=self._genai_config)`
    3. Extract response text: replace `str(response.content).strip()` with `(response.text or "").strip()`
    4. Extract finish_reason: replace `response.response_metadata.get(...)` logic with:
       ```python
       finish_reason = None
       if response.candidates:
           fr = response.candidates[0].finish_reason
           finish_reason = fr.name if fr is not None else None
       ```
    5. Update the COMPLETE_FINISH_REASONS set to `{"STOP", "FINISH_REASON_UNSPECIFIED"}` (remove "STOPPED", "COMPLETE", "COMPLETED", "SUCCESS" which were LangChain interpretations)

    **Do NOT change:**
    - The asyncio.wait_for timeout wrapping — keep it identical
    - Circuit breaker, rate limiter, retry, PII redaction logic — untouched
    - Public method signatures — zero breaking changes
    - Logging and metrics calls — preserve all structured logging
    - The `model_name` property — keep it returning `self.model_name`
  </action>

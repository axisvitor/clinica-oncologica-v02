<objective>
Ensure deferred follow-up sends are retried on failure and day advancement is atomic and verified.

Purpose: Two related silent-failure paths are fixed: (1) When the follow-up system's MessageExecutor fails to send a deferred message, the failure is swallowed and the follow-up is silently dropped. This plan adds a Celery retry task for follow-up sends. (2) When a flow day completes and the day advancement step fails, the flow can end up in a broken state where day_complete=True but current_step was never incremented. This plan makes day advancement atomic with a verification flag.

Output: New Celery task `followup_retry.py`, modified MessageExecutor, atomic day advancement in sequencing, and tests for both.
</objective>

1. In `sequencing.py`, modify the day-completion blocks in `_send_all_sequential`, `_send_wait_each_with_auto_advance`, and `_send_remaining_after_response`:

   Currently each has a pattern like:
   ```python
   step_data["day_complete"] = True
   step_data["awaiting_response"] = False
   # ...
   flow_state.step_data = step_data
   flow_state.last_interaction_at = now_sao_paulo()
   await self.db.commit()
   ```

   Change to:
   ```python
   step_data["day_complete"] = True
   step_data["awaiting_response"] = False
   step_data["day_advance_verified"] = False  # Pre-set to False
   # ... (existing fields)
   flow_state.step_data = step_data
   flow_state.last_interaction_at = now_sao_paulo()
   try:
       await self.db.commit()
       # Verify by re-reading (the commit succeeded, so set verified)
       step_data["day_advance_verified"] = True
       step_data["day_advance_verified_at"] = now_sao_paulo().isoformat()
       flow_state.step_data = step_data
       await self.db.commit()
   except Exception as exc:
       logger.error(
           "Day advancement commit failed — flow may be in inconsistent state",
           extra={
               "patient_id": str(patient.id) if hasattr(patient, 'id') else "unknown",
               "day_number": day_number,
               "flow_kind": flow_kind,
               "error": str(exc),
           },
       )
       raise
   ```

   This ensures `day_advance_verified` is only True after a successful commit.

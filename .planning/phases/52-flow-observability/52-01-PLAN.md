<task type="auto" tdd="true">
  <name>Task 1: Create FlowHealthService with count queries and stall alert</name>
  <files>
    backend-hormonia/app/services/flow/health.py,
    backend-hormonia/app/config/settings/tasks.py,
    backend-hormonia/tests/unit/services/flow/test_flow_health.py
  </files>
  <behavior>
    - get_health_summary(db) returns dict with keys: active, stalled, failed, completed (all int)
    - "active" counts flows where completed_at IS NULL and NOT stalled and NOT failed
    - "stalled" counts flows where completed_at IS NULL, awaiting_response is truthy, and last_interaction_at older than FLOW_STALL_ALERT_HOURS
    - "failed" counts flows where step_data has 'permanently_failed_at' key
    - "completed" counts flows where completed_at IS NOT NULL
    - check_and_fire_stall_alerts(db) emits structured log with patient_id and hours_stuck for each stalled flow
    - check_and_fire_stall_alerts(db) sends POST to FLOW_STALL_ALERT_WEBHOOK_URL if configured, with JSON payload listing stalled flows
    - If FLOW_STALL_ALERT_WEBHOOK_URL is empty/unset, webhook is skipped silently (no error)
    - Stall alert threshold is configurable via TASK_FLOW_STALL_ALERT_HOURS (default 6)
  </behavior>
  <action>
    1. Add two settings to `app/config/settings/tasks.py` under a new FLOW STALL ALERT section:
       - `FLOW_STALL_ALERT_HOURS = int(os.getenv("TASK_FLOW_STALL_ALERT_HOURS", "6"))`
       - `FLOW_STALL_ALERT_WEBHOOK_URL = os.getenv("TASK_FLOW_STALL_ALERT_WEBHOOK_URL", "")`

    2. Create `app/services/flow/health.py` with class `FlowHealthService`:
       - Constructor takes `db: AsyncSession`
       - `async def get_health_summary() -> dict` — runs 4 SQL count queries against PatientFlowState:
         - active: `completed_at IS NULL AND NOT permanently_failed AND NOT stalled` (i.e., not matching stalled or failed criteria)
         - stalled: `completed_at IS NULL AND step_data->>'awaiting_response' IN ('true','True','1','yes') AND last_interaction_at < (now - FLOW_STALL_ALERT_HOURS)` AND Patient.deleted_at IS NULL
         - failed: `completed_at IS NULL AND step_data ? 'permanently_failed_at'` AND Patient.deleted_at IS NULL
         - completed: `completed_at IS NOT NULL`
       - `async def check_and_fire_stall_alerts() -> list[dict]` — finds stalled flows (same query as stalled count but returns patient_id, flow_state_id, last_interaction_at, hours_stuck). For each:
         - Emit structured log: `logger.warning("flow_stall_alert", extra={"patient_id": ..., "flow_state_id": ..., "hours_stuck": ..., "alert_type": "flow_stall"})`
         - If FLOW_STALL_ALERT_WEBHOOK_URL is set, POST JSON payload with `{"stalled_flows": [...], "alert_time": ..., "threshold_hours": ...}` using `httpx.AsyncClient` with 5s timeout. Catch and log any httpx errors without raising.
       - Return list of stalled flow dicts for caller inspection.

    3. Write tests in `tests/unit/services/flow/test_flow_health.py`:
       - Mock AsyncSession with `execute()` returning preset scalar results
       - Test each count category returns correct value
       - Test stall alert emits structured log (capture with caplog)
       - Test stall alert POSTs to webhook URL when configured (mock httpx)
       - Test stall alert skips webhook when URL is empty
  </action>
  <verify>
    <automated>cd backend-hormonia && ./.venv/bin/python -m pytest tests/unit/services/flow/test_flow_health.py -x -q</automated>
  </verify>
  <done>FlowHealthService returns correct counts for active/stalled/failed/completed flows, stall alert fires structured log and optional webhook for stalled flows, all tests pass</done>
</task>

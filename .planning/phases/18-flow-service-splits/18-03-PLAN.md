    Create `app/services/enhanced_flow_engine_pkg/` with five focused modules:

    1) `context.py` — Extract `FlowContext` class (dataclass/plain class) with `to_dict()`, `_calculate_treatment_day()`, and any associated helper functions. Keep the engine-specific FlowContext here — do NOT promote to `flow/types.py` (that has a different Pydantic FlowContext).

    2) `orchestration.py` — Extract `generate_flow_message` and its private helpers: prompt construction, `_get_few_shot_examples`, system prompt building, `_build_personalization_context`.     3) `response_processing.py` — Extract `process_patient_response`, `_calculate_engagement_score`, `_normalize_response_context`, and engagement-related helpers. Define as a mixin class (e.g., `FlowResponseMixin`). If `_calculate_engagement_score` is shared between orchestration and response processing, place it here and let orchestration import from `.response_processing` — OR extract to a `_utils.py` if both modules need it (Claude's discretion per CONTEXT.md).

    4) `conversation.py` — Extract `_get_conversation_history`, `_get_recent_interactions`, `health_check`, and conversation memory helpers. Define as a mixin class (e.g., `FlowConversationMixin`).

    5) `service.py` — Compose `EnhancedFlowEngine` by inheriting from:
       - `FlowOrchestrationMixin` (from `.orchestration`)
       - `FlowResponseMixin` (from `.response_processing`)
       - `FlowConversationMixin` (from `.conversation`)
       - `FlowCore` (from `app.services.flow_core`) — preserving the existing inheritance chain
       Keep `__init__` in this class. Include `get_enhanced_flow_engine()` and `test_enhanced_flow_engine()` factory functions here.

    Create `__init__.py` with exhaustive re-exports and explicit `__all__`:
    - Re-export `EnhancedFlowEngine` from `.service`
    - Re-export `FlowContext` from `.context`
    - Re-export `FlowType` from `app.services.flow.types` (currently imported and re-exported by enhanced_flow_engine.py — transition_handler.py and manual_correction.py depend on this)
    - Re-export `get_enhanced_flow_engine`, `test_enhanced_flow_engine` from `.service`

    Convert `app/services/enhanced_flow_engine.py` into a thin shim with explicit named imports:

    ```python
        EnhancedFlowEngine,

    )

        "EnhancedFlowEngine",
        "FlowContext",
        "FlowType",
        "get_enhanced_flow_engine",
        "test_enhanced_flow_engine",
    ]
    ```

    CRITICAL constraints:
    - `FlowType` is imported from `app.services.flow.types` (NOT defined in enhanced_flow_engine) — the shim must pass it through
    - `EnhancedFlowEngine` MUST still inherit from `FlowCore` — breaking this chain would break all flow operations
    - `FlowContext` in this package is a different class from `app.services.flow.types.FlowContext` — do NOT merge them
    - Mixin classes must NOT define `__init__` (keep `__init__` only in the composed class in `service.py`)
    - Cross-module imports within the package must use relative imports (`from .context import ...`)
    - Keep all method signatures, return types, and business logic identical — structural refactor only
    - The `TYPE_CHECKING` guard for lazy imports in the original file must be preserved in the appropriate split module

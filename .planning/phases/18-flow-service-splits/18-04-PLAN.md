    1) `sequencing.py` — Extract core message send orchestration methods: `send_day_messages`, `_send_all_sequential`, `_send_wait_each_with_auto_advance`, `_send_message_and_wait`, `_send_remaining_after_response`, `_send_flow_message`.     3) `personalization.py` — Extract AI content personalization methods: `_personalize_message_ai`, `_personalize_message_simple`, `_personalization_is_grounded`, `_build_fallback_content`, `_select_template_variation`, `_lightly_rephrase_question`. If `_get_ai_engine()` is also used by sequencing, keep it in service.py or extract a shared helper.

    4) `quiz.py` — Extract `_inject_quiz_link_if_needed` and any related quiz-link logic. Define as a mixin class (e.g., `QuizMixin`). This is a small module — if it ends up very short, that is acceptable (quiz link injection is a distinct concern).

    5) `service.py` — Compose `SequentialMessageHandler` by inheriting from:
       - `SequencingMixin` (from `.sequencing`)
       - `StateMixin` (from `.state`)
       - `PersonalizationMixin` (from `.personalization`)
       - `QuizMixin` (from `.quiz`)
       Keep `__init__` only in the composed class. Include `get_sequential_message_handler()` factory function here.

    Create `__init__.py` with exhaustive re-exports and explicit `__all__`:
    - Re-export `SequentialMessageHandler` from `.service`
    - Re-export `get_sequential_message_handler` from `.service`
    - Re-export any other public symbols that the original file exported

    Convert `app/services/flow/sequential_message_handler.py` into a thin shim with explicit named imports:

    ```python
        SequentialMessageHandler,
        get_sequential_message_handler,
    )
    __all__ = [
        "SequentialMessageHandler",
        "get_sequential_message_handler",
    ]
    ```

    - `SequentialMessageHandler` is a single class with 30+ methods — use the mixin pattern to split methods across modules while keeping a single composed class for callers
    - The `TYPE_CHECKING` guard for `EnhancedFlowEngine` and lazy import pattern (`_get_ai_engine`) must be preserved to avoid circular imports at runtime
    - `AsyncSession` is used throughout — all split modules must preserve async method signatures

    - Callers in `flow_automation.py`, `coordinator.py`, `hive_mind_integration.py`, `response_processor/processor.py`, and `message_handler.py` must continue to work via the shim

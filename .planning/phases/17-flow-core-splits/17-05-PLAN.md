<objective>
Close the final Phase 17 verification gap by fixing the two root causes behind the `422 != 403` assertion failure in `test_idempotency_rbac_denies_other_doctor`:

1. The critical test fixture's `client` does not override `get_async_db`, so the endpoint's `AsyncSession` operates on a different connection/transaction than the test's sync `db_session`. The idempotency DB query cannot see test-committed data, so the RBAC check is never reached.
2. The patient validation services use legacy `session.query()` (SQLAlchemy 1.x API) which raises `AttributeError` on `AsyncSession` objects, causing a caught exception that manifests as a validation error (422) instead of the expected authorization denial (403).

Purpose: With both fixes applied, the idempotency key lookup finds the test patient, triggers `ensure_patient_access`, and correctly raises 403. The validation services also become compatible with both sync and async sessions for any code path that reaches them.
Output: Updated test fixture, two patched service files, and documented full-suite evidence.
</objective>

  <action>
    This task fixes both root causes of the 422-vs-403 failure. Both must land together because either fix alone is insufficient.

    **Part A: Override `get_async_db` in the `client` fixture (conftest.py)**

    The `client` fixture currently only overrides `get_db` (sync). The `create_patient` endpoint at `crud.py:576` depends on `get_async_db` (returns `AsyncSession`), so the endpoint operates on a completely different database connection that cannot see data committed inside the test's sync `db_session` transaction.

    Implementation:
    1. Add a `SyncToAsyncSessionAdapter` class inside `conftest.py` (above the `client` fixture) that wraps a sync `Session` to present an async-compatible interface. The adapter needs:
       - `async def execute(self, statement, *args, **kwargs)` -> delegates to `self._sync_session.execute(statement, *args, **kwargs)` and returns the result.
       - `async def commit(self)` -> delegates to `self._sync_session.flush()` (the outer test transaction handles commit/rollback).
       - `async def rollback(self)` -> no-op (outer transaction handles this).
       - `async def close(self)` -> no-op (fixture handles cleanup).
       - `async def __aenter__(self)` -> returns `self`.
       - `async def __aexit__(self, *args)` -> no-op.
       - `def __getattr__(self, name)` -> delegates attribute access to `self._sync_session` for any attribute not explicitly defined (e.g., `.scalars()`).
    2. In the `client` fixture, import `get_async_db` from `app.database` (add to `app_modules` or import directly).
    3. Create an async generator override for `get_async_db` that yields a `SyncToAsyncSessionAdapter(db_session)` instance.
    4. Register the override: `app_instance.dependency_overrides[get_async_db] = override_get_async_db`.
    5. Ensure the override is cleared in the `finally`/cleanup block (already handled by `app_instance.dependency_overrides.clear()`).

    CRITICAL: The adapter must also support `.scalars()` access on the result of `execute()`. Since `self._sync_session.execute(stmt)` returns a `Result` object that already has `.scalars()`, this works transparently as long as `execute` returns the real result object.

    **Part B: Fix `_validate_doctor_exists` in validation_service.py (line 201-208)**

    Replace the legacy `self.db.query(User).filter(User.id == doctor_id).first()` with SQLAlchemy 2.0-style:
    ```python
    from sqlalchemy import select
    result = self.db.execute(select(User).filter(User.id == doctor_id))
    doctor = result.scalars().first()
    ```

    This works on both sync `Session` and the `SyncToAsyncSessionAdapter` because both implement `.execute()` returning a `Result` with `.scalars()`.

    Add a `# TODO(async-migration): converted from .query() to select() for AsyncSession compat` annotation following the codebase convention.

    **Part C: Fix `_check_duplicate_by_hashed_field` in sync_service.py (line 55-80)**

    Replace the legacy query-builder pattern:
    ```python
    query = self.db.query(Patient).filter(...)
    if doctor_id:
        query = query.filter(...)
    if exclude_patient_id:
        query = query.filter(...)
    return query.first()
    ```

    With SQLAlchemy 2.0-style:
    ```python
    from sqlalchemy import select
    stmt = select(Patient).filter(patient_field == field_hash, Patient.deleted_at.is_(None))
    if doctor_id:
        stmt = stmt.filter(Patient.doctor_id == doctor_id)
    if exclude_patient_id:
        stmt = stmt.filter(Patient.id != exclude_patient_id)
    result = self.db.execute(stmt)
    return result.scalars().first()
    ```

    Add a `# TODO(async-migration): converted from .query() to select() for AsyncSession compat` annotation.

    IMPORTANT CONSTRAINTS:
    - Do NOT change any other methods in these files.
    - Do NOT modify the test logic in `test_patient_security_fixes.py`.
    - The `select` import may already exist in some files; check before adding a duplicate.
    - Keep the error handling (`try/except`) wrappers intact in both service methods.

<objective>
Add three WuzAPI environment variables to IntegrationsSettings with a hard-fail startup validator for the token, update the boolean parse list, and replace Evolution API entries in .env.example with WuzAPI entries.

Purpose: Ensure that WuzAPIClient can read its configuration from the standard settings object, that the application refuses to start without a token (preventing silent fallback in production), and that .env.example guides developers to the correct env var names.
Output: IntegrationsSettings has three new fields, Settings.parse_env_values has the mock boolean, .env.example has WuzAPI block instead of Evolution block.
</objective>

Key decisions from STATE.md (LOCKED -- do not change):
- Evolution settings fields are NOT removed from IntegrationsSettings in this phase (Phase 37 does that)
- Hard-fail means ValueError at pydantic model instantiation -- not a lifespan check
- Test environments (PYTEST_CURRENT_TEST, TESTING=1, APP_ENVIRONMENT=test/testing) are exempt from the hard-fail

CRITICAL anti-patterns to avoid:
- Do NOT remove WHATSAPP_EVOLUTION_* fields from IntegrationsSettings -- they are still imported by lifespan.py and whatsapp.py monitoring router
- Do NOT use a default value like `str = Field(default="")` for the token -- this would allow silent empty-string startup
- Do NOT access `self.APP_ENVIRONMENT` before it is populated in the model_validator -- use `mode="after"` so `self` is fully initialized

    3. Add the hard-fail model_validator INSIDE the `IntegrationsSettings` class, after the new fields and before the `# WhatsApp Integration Configuration` comment. Place it after all field declarations:

       ```python
       @model_validator(mode="after")
       def validate_wuzapi_token(self) -> "IntegrationsSettings":
           """Hard-fail at startup if WHATSAPP_WUZAPI_TOKEN is missing (CFG-02).

           Application must refuse to start without the token in non-test environments.
           Silent fallback is explicitly prohibited.
           """
           import os

           is_test = bool(
               os.getenv("PYTEST_CURRENT_TEST")
               or os.getenv("TESTING") == "1"
               or self.APP_ENVIRONMENT.lower() in ("test", "testing")
           )
           if is_test:
               return self

           token = self.WHATSAPP_WUZAPI_TOKEN
           if not token or not token.strip():
               raise ValueError(
                   "\n" + "=" * 70 + "\n"
                   "STARTUP VALIDATION FAILED: WHATSAPP_WUZAPI_TOKEN is required.\n"
                   "=" * 70 + "\n"
                   "Set WHATSAPP_WUZAPI_TOKEN in your .env file or environment.\n"
                   "This token authenticates all WuzAPI API calls.\n"
                   "Obtain it from your WuzAPI instance configuration.\n"
                   "=" * 70
               )
           return self
       ```

       NOTE: The validator uses `mode="after"` so that `self.APP_ENVIRONMENT` is already populated. It checks `PYTEST_CURRENT_TEST`, `TESTING`, and `APP_ENVIRONMENT` to avoid breaking test suites.

    4. Do NOT move, remove, or modify any existing `WHATSAPP_EVOLUTION_*` field.

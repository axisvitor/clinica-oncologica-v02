<objective>
Standardize pause detection so that `state_data.paused` is the single source of truth across the daily processor, FlowCore, and FlowManagementService. Currently FlowCore writes pause info to `step_data["paused"]` (as a dict) while FlowManagementService writes to `state_data["paused"]` (as a boolean), and the daily processor reads `step_data.paused`. This mismatch means paused flows can still be processed by the daily job.

Purpose: A paused patient flow must stay paused — the daily processor must reliably skip it.
Output: Consistent pause field across all three components with tests proving the contract.
</objective>

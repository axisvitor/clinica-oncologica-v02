<objective>
Rewrite the `resume_paused_flows` Celery Beat task to check `state_data.auto_resume_at` timestamps instead of using a blanket 48-hour `updated_at` heuristic. Currently, the task resumes any flow paused for more than 48 hours regardless of whether auto-resume was requested. After this fix, only flows with an explicit `auto_resume_at` timestamp that has expired will be auto-resumed. Indefinite pauses stay paused.

Purpose: Auto-resume works as the doctor intended — only when they set a resume time.
Output: Fixed auto-resume task with proper timestamp checking and matching tests.
</objective>

    **Rewrite `resume_paused_flows` in flow_automation.py (line ~289-357):**

    1. Replace the SQL query that uses `pfs.updated_at < NOW() - INTERVAL '48 hours'` with a query that:
       - Selects flows where `pfs.status = 'paused'`
       - AND `pfs.state_data->>'auto_resume_at' IS NOT NULL` (only flows with auto-resume set)
       - AND `(pfs.state_data->>'auto_resume_at')::timestamptz <= NOW()` (expired auto-resume timestamp)
       - AND `p.deleted_at IS NULL`
       - Keep `FOR UPDATE OF pfs SKIP LOCKED` for concurrency safety
       - Keep `LIMIT 50` for batch safety

    2. For each matched flow, instead of calling `flow_engine.resume_patient_flow(flow_row.id)` (which expects patient_id), use FlowManagementService for consistency with the state_data.paused field aligned in Plan 01:
       ```python
       from app.repositories.flow import FlowStateRepository
       from app.services.flow_management import FlowManagementService

       flow_repo = FlowStateRepository(db)
       mgmt_service = FlowManagementService(flow_repo, db)
       await mgmt_service.resume_patient_flow(patient_id=flow_row.patient_id)
       ```

    3. Add audit logging for auto-resume events using the existing structured logging pattern:
       ```python
       logger.info(
           "Auto-resumed flow",
           extra={
               "patient_id": str(flow_row.patient_id),
               "flow_id": str(flow_row.id),
               "auto_resume_at": flow_row.state_data.get("auto_resume_at") if flow_row.state_data else None,
               "action": "auto_resume",
               "actor": "celery_beat",
           },
       )
       ```

    4. Handle the case where `resume_patient_flow` raises `FlowStateConflictError("Flow is not currently paused")` — this means the flow was already resumed manually. Log a warning and continue (don't count as error).

Change schedule to every 1 hour (`3600.0`) — this is between the 15-min check-pending and 6-hour current value, and matches the granularity needed for timestamp-based auto-resume.
    - Add a descriptive comment: `# Every hour — checks auto_resume_at timestamps on paused flows`

    AVOID: Do NOT use `flow_engine.resume_patient_flow(flow_row.id)` — that's the FlowCore async path which writes to step_data. Use FlowManagementService which writes to state_data (aligned in Plan 01).

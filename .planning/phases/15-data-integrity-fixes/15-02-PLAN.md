  <action>
    1. In `trigger_service.py`, modify `_get_or_create_monthly_template` to return `Optional[QuizTemplate]` instead of raising on failure. Currently the method creates a template if none exists — keep that behavior, but wrap the entire method in a try/except that returns `None` on failure instead of propagating the exception. The current method already has a try/except but raises in the else path after template creation fails.

    2. In `trigger_service.py`, modify `_trigger_patient_quiz` (the main entry point at line ~259):
       - After `template = await self._get_or_create_monthly_template(quiz_info)`, add a None check:
       ```python
       if template is None:
           logger.warning(
               "Quiz template not found for patient %s, flow_type=%s, cycle=%s. "
               "Sending message without quiz link.",
               patient_id,
               flow_state.flow_type,
               quiz_info.get("monthly_cycle"),
               extra={
                   "patient_id": str(patient_id),
                   "flow_context": quiz_info,
                   "fallback_reason": "quiz_template_missing",
               },
           )
           # Mark flow state with fallback metadata
           flow_state.state_data = flow_state.state_data or {}
           flow_state.state_data["quiz_template_missing_fallback"] = True
           flow_state.state_data["quiz_fallback_at"] = now_sao_paulo().isoformat()
           flow_state.state_data["quiz_state"] = "skipped_no_template"
           self.db.commit()

           return {
               "success": False,
               "patient_id": str(patient_id),
               "error": "quiz_template_not_found",
               "fallback_applied": True,
               "message": "Quiz skipped - template not available. Message sent without quiz link.",
           }
       ```

    3. In `monthly_quiz_message_integration.py`, in `send_quiz_link` method, after the patient lookup and before `MonthlyQuizLinkCreate`, add a defensive check:
       ```python
       # Validate quiz template exists before creating link
       from app.repositories.quiz import QuizTemplateRepository
       template_repo = QuizTemplateRepository(self.db)
       template = template_repo.get(quiz_template_id)
       if not template:
           logger.warning(
               "Quiz template %s not found for patient %s. Skipping quiz link creation.",
               quiz_template_id, patient_id,
           )
           return {
               "quiz_session_id": None,
               "token": None,
               "link_url": None,
               "expires_at": None,
               "message_sent": False,
               "fallback_reason": "template_not_found",
           }
       ```
       This prevents the `NotFoundError` from `create_quiz_link` from crashing the caller.

    4. Do NOT create a separate message template for the fallback. Do NOT add real-time staff alerts. Per user decision, this surfaces in existing monitoring/dashboard via the warning log.

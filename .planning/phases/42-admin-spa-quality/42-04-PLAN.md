<task type="auto">
  <name>Task 1: Run knip audit and remove confirmed-unused npm packages</name>
  <files>frontend-hormonia/package.json</files>
  <action>
    **Step 1 — Run knip:**
    ```bash
    cd frontend-hormonia && npx knip --include-entry-exports 2>&1
    ```

    Capture the output. knip will report:
    - Unused dependencies (packages in dependencies/devDependencies not imported)
    - Unused exports (exported symbols not imported anywhere)
    - Unused files (files not imported from any entry point)

    Focus ONLY on unused dependencies for this task.

    **Step 2 — Triage each reported unused dependency:**

    For each package knip reports as unused:
    1. Search for actual usage: `grep -rn "from '${pkg}'" src/` and `grep -rn "import('${pkg}')" src/`
    2. Check if it's a shadcn/ui peer dependency needed at runtime (many @radix-ui packages are used via shadcn components in `src/components/ui/`)
    3. Check if it's used in config files that knip doesn't trace (vite.config.ts, vitest config, playwright config)

    **Classification:**
    - CONFIRMED UNUSED: No import found anywhere in src/, config files, or test files -> REMOVE
    - FALSE POSITIVE: Used via dynamic import, re-export, or config file -> KEEP
    - UNCERTAIN: Mark for human review in the checkpoint

    **Step 3 — Remove confirmed unused packages:**
    ```bash
    cd frontend-hormonia && npm uninstall <package1> <package2> ...
    ```

    **Step 4 — Verify after removal:**
    ```bash
    cd frontend-hormonia && npx tsc --noEmit && npx eslint . --max-warnings 50 && npm run build
    ```

    If `tsc --noEmit` or `npm run build` fails after removal, the package was NOT actually unused — reinstall it and remove from the CONFIRMED UNUSED list.

    **CRITICAL:** Do NOT blindly trust knip output. Verify each package manually before uninstalling. Dynamic imports (`import('recharts')`), CSS module imports, and @types/* packages are common false positives.
  </action>
  <verify>
    <automated>cd frontend-hormonia && npx tsc --noEmit && npx eslint . --max-warnings 50</automated>
  </verify>
  <done>
    - All confirmed-unused packages removed from package.json
    - tsc --noEmit exits 0 (no type errors from removal)
    - eslint exits with 0 errors
    - Build still succeeds (npm run build or vite build)
  </done>
</task>

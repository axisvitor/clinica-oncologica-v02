    **Step 1 — Determine existing code style:**

    Before choosing prettier config values, check a sample of .tsx files to determine the dominant style:
    - `head -20 frontend-hormonia/src/features/admin/tabs/AdminSettingsTab.tsx` — check semi usage
    - `head -20 frontend-hormonia/src/components/hive-mind/AgentSwarm.tsx` — check semi usage
    - `head -20 frontend-hormonia/src/lib/api-client/core.ts` — check semi usage

    Determine: Do most files use semicolons or not? Check for `import x from 'y';` (with semi) vs `import x from 'y'` (no semi). The research suggests mixed usage — use `semi: false` only if the majority clearly omits them; otherwise default to `semi: true` to match the more common TypeScript convention.

    **Step 2 — Install packages:**
    ```bash
    cd frontend-hormonia && npm install --save-dev prettier eslint-config-prettier
    ```

    **Step 3 — Create .prettierrc:**
    ```json
    {
      "semi": <true or false based on Step 1>,
      "singleQuote": true,
      "tabWidth": 2,
      "trailingComma": "es5",
      "printWidth": 100,
      "plugins": []
    }
    ```

    **Step 4 — Create .prettierignore:**
    ```
    dist/
    node_modules/
    coverage/
    playwright-report/
    *.min.js
    ```

    **Step 5 — Update eslint.config.js:**
    Add `import prettierConfig from 'eslint-config-prettier'` at the top.
    Add `prettierConfig` as the LAST item in the `tseslint.config(...)` array — AFTER the test files override block. This disables all ESLint formatting rules that conflict with Prettier.

    The final structure should be:
    ```javascript
    import prettierConfig from 'eslint-config-prettier'
    // ... other imports ...

    export default tseslint.config(
      { ignores: [...] },
      { extends: [...], files: [...], rules: {...} },
      { files: ['**/*.test.*'], rules: {...} },
      prettierConfig  // MUST be last
    )
    ```

    **Step 6 — Add npm scripts to package.json:**
    Add to the "scripts" section:
    ```json
    "format": "prettier --write 'src/**/*.{ts,tsx}'",
    "format:check": "prettier --check 'src/**/*.{ts,tsx}'"
    ```

    **Step 7 — Run Prettier on all source files:**
    ```bash
    cd frontend-hormonia && npx prettier --write 'src/**/*.{ts,tsx}'
    ```

    **Step 8 — Verify no ESLint/Prettier conflicts:**
    ```bash
    cd frontend-hormonia && npx eslint .
    ```
    If eslint reports errors on lines that Prettier just formatted, eslint-config-prettier is not properly integrated — verify it's the last item in the config array.

    **IMPORTANT:** Do NOT run `prettier --write .` (without the src/ glob) — it will try to format files in dist/, node_modules/, etc. Always use the glob pattern or rely on .prettierignore.

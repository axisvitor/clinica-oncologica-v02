    **routes.py changes (2 edits):**

    1. Rewrite the `sync_contacts` handler (lines 205-221) to immediately raise HTTPException(501). The new handler:
       ```python
       @router.post("/contacts/{instance_name}/sync")
       async def sync_contacts(instance_name: str):
           """Contacts sync is not supported by WuzAPI.

           WuzAPI (whatsmeow-based) does not expose a contacts API equivalent to
           Evolution API. This endpoint is retained for API compatibility and returns
           HTTP 501 to avoid silent failures from the previous stub implementation.
           """
           raise HTTPException(
               status_code=501,
               detail=(
                   "Contacts sync is not supported by WuzAPI. "
                   "This operation has no equivalent in the current WhatsApp provider."
               ),
           )
       ```
       Remove the `background_tasks: BackgroundTasks` parameter and the `message_service` Depends injection from the signature. The handler no longer needs either.

    2. Remove `BackgroundTasks` from the import on line 8 since it is no longer used anywhere in routes.py. The import becomes:
       `from fastapi import APIRouter, Depends, HTTPException, Query`

    **Why 501 not route removal:** Per research decision and Phase 36 decision log — the route may appear in client code or API documentation. A 501 is self-documenting ("server understands the request but doesn't implement it"), while a 404 from a removed route looks like a bug.

<objective>
Implement the LGPD batch re-encryption Celery task that enables cryptographic key rotation for patient PII fields (CPF, email, phone).

Purpose: LGPD Art. 46 requires the ability to rotate encryption keys. The `UnifiedEncryptionService.rotate_encryption_key()` method is currently a stub with a `# TODO: Implement batch re-encryption` comment. This plan replaces that stub with a real Celery task that processes patients in chunks with Redis-based idempotency, enabling safe key rotation without data loss or corruption.

Output: A fully operational `batch_reencrypt_patients` Celery task, registered in the task system, with a test verifying idempotency.
</objective>

Create `backend-hormonia/app/tasks/lgpd/` package with `__init__.py` and `reencrypt_patients.py`.

**`reencrypt_patients.py`** must implement:

1. **Helper `_make_encryption_service(master_key: str) -> UnifiedEncryptionService`:**
   - Create `UnifiedEncryptionService(auto_initialize=False)`
   - Derive PHI key via `svc._derive_key(master_key, b"hormonia_unified_salt_2025")` and set `svc._keys["phi"]`
   - Derive quiz key via `svc._derive_fernet_key(master_key)` and set `svc._keys["quiz"]`
   - Call `svc._initialize_algorithms()` then `svc._initialize_field_encryptors()`
   - Return the service instance

2. **Helper `_reencrypt_patient(patient, old_svc, new_svc) -> bool`:**
   - For each encrypted field (`cpf_encrypted`, `email_encrypted`, `phone_encrypted`):
     - If field is not None, decrypt with `old_svc.decrypt_cpf/email/phone()`
     - If plaintext is truthy, encrypt with `new_svc.encrypt_cpf/email/phone()`
     - Set both `*_encrypted` and `*_hash` columns on the patient object
   - Return True if any field was changed
   - On decryption error, log with patient ID and re-raise (let the caller decide skip vs abort)

3. **Redis idempotency helpers:**
   - `_already_processed(redis_client, job_id, patient_id) -> bool` using `sismember` on key `lgpd:reencrypt:completed_ids:{job_id}`
   - `_mark_processed(redis_client, job_id, patient_id)` using `sadd` + `expire` (7 days TTL)

4. **Main task `batch_reencrypt_patients`:**
   - Decorator: `@celery_app.task(bind=True, base=DatabaseTask, name="lgpd.batch_reencrypt_patients", queue="celery", max_retries=3, default_retry_delay=30, retry_backoff=True, retry_backoff_max=300)`
   - Parameters: `old_key_env_var: str = "PHI_ENCRYPTION_KEY_PREVIOUS"`, `new_key_env_var: str = "PHI_ENCRYPTION_KEY"`, `job_id: str`, `chunk_size: int = 100`, `max_patients: int = 10000`
   - Read actual keys from `os.environ[old_key_env_var]` and `os.environ[new_key_env_var]` (NOT as direct args -- secrets must not appear in Celery result backend or broker logs)
   - Assert both keys are present; assert `HASH_SALT` env var exists and log a warning that HASH_SALT must NOT be changed during key rotation
   - Create `old_svc` and `new_svc` via `_make_encryption_service()`
   - Get Redis sync client via `get_redis_manager().get_sync_client()`
   - Chunked loop: `offset=0`, query `Patient` where `deleted_at.is_(None)` ordered by `Patient.id`, `.offset(offset).limit(chunk_size)`, break when batch is empty or `processed >= max_patients`
   - Per-patient: check `_already_processed()`, if not processed call `_reencrypt_patient()`, on success call `_mark_processed()`, on exception increment `errors` counter and log (do NOT re-raise per patient -- continue to next patient; the error is logged and counted)
   - Commit per chunk via `db.commit()`
   - Use `get_scoped_session()` from `app.database` for the DB session (Celery tasks use sync sessions)
   - Track and return stats dict: `{"processed": N, "re_encrypted": N, "skipped": N, "errors": N, "has_more": bool}`
   - `has_more` is True if the last batch was full AND `processed < max_patients` is still false (i.e., stopped at max_patients limit but more records may exist)
   - Update task progress via `self.update_state(state="PROGRESS", meta={"processed": processed, "re_encrypted": re_encrypted})` after each chunk

**`__init__.py`:**
```python
from .reencrypt_patients import batch_reencrypt_patients  # noqa: F401

__all__ = ["batch_reencrypt_patients"]
```

Follow existing conventions: `from __future__ import annotations`, `logger = logging.getLogger(__name__)`, module docstring explaining purpose.

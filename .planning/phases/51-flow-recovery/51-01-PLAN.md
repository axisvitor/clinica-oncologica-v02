<objective>
Detect stuck patient flows automatically and recover them via periodic Celery Beat task.

Purpose: Patients whose flows are stuck in `awaiting_response` for hours are currently invisible. This plan adds a periodic detector that finds them and attempts auto-recovery (re-send prompt or advance day) before the patient or operator notices.

Output: `recovery.py` service module with detection + recovery logic, `stuck_detection.py` Celery task, Beat schedule entry, and configurable settings. Fully tested.
</objective>

  <behavior>
    - find_stuck_flows returns flows where awaiting_response is truthy AND last_interaction_at is older than threshold
    - find_stuck_flows returns empty list when no flows are stuck
    - find_stuck_flows respects configurable threshold hours via STUCK_FLOW_DETECT_HOURS setting
    - find_stuck_flows caps results at 100 per cycle (safety limit)
    - determine_recovery_action returns "advance_day" when day_complete=True and day_advance_verified=False
    - determine_recovery_action returns "resend_prompt" when day_complete=False (normal stuck)
    - determine_recovery_action returns "resend_prompt" when day_complete=True and day_advance_verified=True (edge case)
    - attempt_recovery skips flows that have recovery_attempts >= STUCK_FLOW_MAX_RECOVERY_ATTEMPTS
    - attempt_recovery increments recovery_attempts in step_data and sets last_recovery_at timestamp
    - attempt_recovery uses Redis idempotency key recovery:{flow_state_id} with 15-min TTL to prevent double-recovery
    - attempt_recovery re-checks awaiting_response before acting (race condition guard)
  </behavior>
  <action>
    1. Add settings to `app/config/settings/tasks.py` under a new STUCK FLOW DETECTION section:
       - `STUCK_FLOW_DETECT_HOURS = int(os.getenv("TASK_STUCK_FLOW_DETECT_HOURS", "4"))` -- hours before a flow is considered stuck
       - `STUCK_FLOW_MAX_RECOVERY_ATTEMPTS = int(os.getenv("TASK_STUCK_FLOW_MAX_RECOVERY_ATTEMPTS", "3"))` -- max auto-recovery attempts per flow
       - `STUCK_FLOW_RECOVERY_IDEMPOTENCY_TTL = int(os.getenv("TASK_STUCK_FLOW_RECOVERY_IDEMPOTENCY_TTL_SECONDS", "900"))` -- 15 min TTL for idempotency key

    2. Create `app/services/flow/recovery.py` with:
       - `find_stuck_flows(db: Session, threshold_hours: int = STUCK_FLOW_DETECT_HOURS) -> list[PatientFlowState]`:
         Query PatientFlowState joined with Patient, filtering:
         (a) completed_at IS NULL
         (b) Patient.deleted_at IS NULL
         (c) step_data->>'awaiting_response' IN ('true', 'True', '1', 'yes')
         (d) last_interaction_at < (now - threshold_hours) OR last_interaction_at IS NULL
         Order by last_interaction_at ASC NULLS FIRST, limit 100.

       - `determine_recovery_action(step_data: dict) -> str`:
         If day_complete is truthy and day_advance_verified is falsy: return "advance_day".
         Otherwise: return "resend_prompt".

       - `attempt_recovery(db: Session, flow_state: PatientFlowState, redis_client) -> dict`:
         (a) Check recovery_attempts in step_data vs STUCK_FLOW_MAX_RECOVERY_ATTEMPTS. If exceeded, return {"status": "max_attempts_exceeded"}.
         (b) Check Redis idempotency key `recovery:{flow_state.id}`. If exists, return {"status": "already_recovering"}.
         (c) Set idempotency key with RECOVERY_IDEMPOTENCY_TTL.
         (d) Re-read flow state from DB and re-check is_awaiting_response. If no longer awaiting, return {"status": "no_longer_stuck"}.
         (e) Increment recovery_attempts and set last_recovery_at in step_data. Commit with optimistic version check.
         (f) Determine action via determine_recovery_action.
         (g) For "resend_prompt": enqueue `retry_failed_flow_send` task (reuse Phase 50 pattern). For "advance_day": call FlowManagementService.advance_patient_flow() via sync bridge.
         (h) Return {"status": "recovered", "action": action, "flow_state_id": str(flow_state.id)}.

       Use `from app.services.flow.flags import is_awaiting_response` for Python-level checks.
       Use `from app.utils.timezone import now_sao_paulo` for timestamps.
       Import PatientFlowState from `app.models.flow`, Patient from `app.models.patient`.

    3. Write tests in `tests/unit/services/flow/test_flow_recovery.py`:
       - Test find_stuck_flows with mocked DB query returning stuck flows
       - Test find_stuck_flows with empty result
       - Test determine_recovery_action for each branch
       - Test attempt_recovery max attempts exceeded
       - Test attempt_recovery idempotency key already set
       - Test attempt_recovery race condition (no longer stuck on re-check)
       - Test attempt_recovery successful resend path
       - Test attempt_recovery successful advance path

    CRITICAL: Do NOT use `redis.keys()` anywhere -- use `scan_iter(match=..., count=100)` per project rules. For the idempotency key, use simple `get`/`set` with TTL, no scanning needed.
  </action>

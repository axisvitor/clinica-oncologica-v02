  <behavior>
    - POST /{patient_id}/reset clears awaiting_response, context_mismatch_count, and pending_response_context from step_data, bumps version
    - POST /{patient_id}/advance calls day advancement for the patient's active flow, returns new day number
    - POST /{patient_id}/unstick clears awaiting_response, recovery_attempts, last_recovery_at from step_data, bumps version
    - GET /failed returns paginated list of flows with delivery_failures or last_mismatch_reset_at in step_data
    - All endpoints require admin auth (get_admin_user dependency)
    - All mutating endpoints audit-log the action via log_admin_extension_action
    - All mutating endpoints return 404 if no active flow exists for patient_id
    - GET /failed supports limit and offset query params with defaults (limit=50, offset=0)
  </behavior>
  <action>
    1. Add schemas to `app/schemas/v2/admin_extensions.py` (append to existing file):
       ```python
       # === Flow Ops Schemas ===

       class FlowOpsResetResponse(BaseModel):
           patient_id: UUID
           flow_state_id: UUID
           action: str = "reset"
           cleared_fields: list[str]

       class FlowOpsAdvanceResponse(BaseModel):
           patient_id: UUID
           flow_state_id: UUID
           action: str = "advance"
           new_day: int

       class FlowOpsUnstickResponse(BaseModel):
           patient_id: UUID
           flow_state_id: UUID
           action: str = "unstick"
           cleared_fields: list[str]

       class FailedFlowOperation(BaseModel):
           flow_state_id: UUID
           patient_id: UUID
           patient_name: str | None
           current_step: int | None
           failure_type: str  # "delivery_failure" | "mismatch_reset"
           failure_details: dict
           updated_at: datetime | None

       class FailedFlowOperationsResponse(BaseModel):
           items: list[FailedFlowOperation]
           total: int
           limit: int
           offset: int
       ```

    2. Create `app/api/v2/routers/admin_extensions/flow_ops.py`:

       Follow the exact pattern from `dlq.py`:
       - Import `get_admin_user`, `log_admin_extension_action` from `.dependencies`
       - Import `get_async_db` from `app.core.database.async_engine`
       - Import `get_request_context` from `app.utils.request_context`
       - Use `@limiter.limit("30/minute")` on mutating endpoints, `@limiter.limit("60/minute")` on GET

       **POST `/{patient_id}/reset`:**
       - Query PatientFlowState via async session: `SELECT ... WHERE patient_id = :patient_id AND completed_at IS NULL`
       - If not found, raise HTTPException(404)
       - Clear from step_data: `awaiting_response`, `context_mismatch_count`, `pending_response_context`
       - Bump `version += 1`
       - Commit
       - Audit log via `log_admin_extension_action(audit_service, "flow_reset", admin_user, context, {"patient_id": str(patient_id)})`
       - Return FlowOpsResetResponse

       **POST `/{patient_id}/advance`:**
       - Find active flow for patient
       - If not found, raise HTTPException(404)
       - Determine target_day = current_day + 1 from step_data
       - Use `advance_day_atomic()` via async call (it's already async)
       - Audit log
       - Return FlowOpsAdvanceResponse with new_day

       **POST `/{patient_id}/unstick`:**
       - Find active flow for patient
       - If not found, raise HTTPException(404)
       - Clear from step_data: `awaiting_response`, `recovery_attempts`, `last_recovery_at`, `context_mismatch_count`
       - Bump `version += 1`
       - Commit
       - Audit log
       - Return FlowOpsUnstickResponse

       **GET `/failed`:**
       - Accept query params: `limit: int = 50`, `offset: int = 0`
       - Query PatientFlowState joined with Patient, filtering:
         `step_data.has_key('delivery_failures') OR step_data.has_key('last_mismatch_reset_at')`
       - Filter `Patient.deleted_at IS NULL`
       - Order by `PatientFlowState.updated_at DESC`
       - Apply limit/offset
       - Count total matching rows (separate count query)
       - Map each result to FailedFlowOperation, setting failure_type based on which marker exists
       - Return FailedFlowOperationsResponse

       CRITICAL: Use async session throughout (not sync). For `advance_day_atomic()`, call it directly since it's already async. For step_data writes, use raw async SQL or `db.execute()` pattern from the DLQ router.

    3. Register in `app/api/v2/routers/admin_extensions/__init__.py`:

       - Add: `router.include_router(flow_ops_router, prefix="/flow-ops", tags=["Admin Extensions - Flow Ops"])`

# T04: Added the explicit M015 session seam probe entrypoint, corrected the probe to exercise the users session API surface, and restored the root-level harness verification gate.

**Added the explicit M015 session seam probe entrypoint, corrected the probe to exercise the users session API surface, and restored the root-level harness verification gate.**

## What Happened

Built `scripts/security/m015-runtime/session_seam.py` as the explicit session-probe entrypoint while preserving the Taskiq worker registration module used by the real worker. Updated Docker Compose so `session-probe` runs the new entrypoint and still mounts the worker task module for both the probe and `app.tasks` import paths. Corrected the session probe to drive `/api/v2/users/me` for dependency-level cookie auth and `/api/v2/users/sessions/{session_id}` for explicit revocation invalidation. Added negative legacy transport proof for `X-Session-ID` and bearer-only requests, expanded the redaction-validated evidence shape with versions, failure classes, non-goals, and no-raw-cookie/session flags, and refreshed the README with session seam usage, recovery, evidence, and non-goal notes. Added static contract coverage for the session seam entrypoint/endpoints/evidence shape and a root-level pytest wrapper so the automated `tests/security/test_m015_runtime_harness.py` gate exercises the canonical backend harness tests instead of failing on a missing path.

## Verification

Verified syntax, static runner contracts, backend runtime harness contracts, S02 session auth contracts, and the previously failing root-level pytest invocation. The final exact task verification command passed with 44 passing tests and 1 expected skip for runtime artifacts that are only produced by the Docker gate; the root-level gate passed with 23 passing tests and 1 expected skip.

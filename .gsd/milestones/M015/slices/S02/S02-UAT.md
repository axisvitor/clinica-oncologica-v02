# S02: Cross-Process Session Revocation + Queue Worker Proof — UAT

**Milestone:** M015
**Written:** 2026-05-14

## UAT Type

- UAT mode: live-runtime
- Why this mode is sufficient: S02 is a backend runtime security seam; the proof must run through separate PostgreSQL, Dragonfly, FastAPI, and Taskiq worker services, not a browser or in-process test client.

## Preconditions

- Docker is available locally.
- The repository contains the S01/S02 M015 harness assets.
- No real provider credentials or patient data are required; all runtime data is synthetic.

## Smoke Test

Run `./scripts/security/verify-m015-runtime-security.sh --seam session` from the repository root and confirm it exits 0 with teardown complete.

## Test Cases

### 1. Current and cache-miss staff sessions are accepted safely

1. Run `./scripts/security/verify-m015-runtime-security.sh --seam session`.
2. Inspect `backend-hormonia/docs/reports/security/m015/session-seam-summary.md`.
3. **Expected:** current cookie-backed session is `200/allowed`, cache miss falls back to PostgreSQL and rehydrates Dragonfly.

### 2. Revoked and expired sessions fail closed despite stale cache

1. Run the session seam.
2. Inspect the session evidence summary.
3. **Expected:** revoked and expired stale-cache cases return `401` with `invalid_or_expired_session`.

### 3. Explicit revocation invalidates cache and worker re-checks DB

1. Run the session seam.
2. Inspect the explicit revocation and worker rows in the summary.
3. **Expected:** explicit revocation reports cache after `missing`, follow-up access is denied, and the Taskiq worker reports `denied_after_db_recheck` with reason `revoked_or_expired`.

## Edge Cases

### Legacy transports remain rejected

1. Run the session seam.
2. Inspect legacy header/Bearer transport outcomes.
3. **Expected:** `X-Session-ID` and bearer-only requests return `401` because a canonical session cookie is required.

## Failure Signals

- Runner exits non-zero.
- Evidence is missing or redaction validation fails.
- Any revoked/expired/legacy/worker-denial case returns allowed.
- Teardown is not marked complete.

## Requirements Proved By This UAT

- R013 — multi-process staff-session revocation/cache-fallback behavior is proven through the runtime seam.
- R014 — the runtime harness now includes session/cache/worker validation beyond DB substrate.
- R015 — proof uses synthetic-only data and no live provider credentials.
- R017 — evidence omits raw cookies, session IDs, DSNs, provider payloads, PHI-shaped values, and host paths.
- R018 — downstream non-goals and remaining seams are listed rather than silently dropped.

## Not Proven By This UAT

- WuzAPI/Gemini provider stub behavior; assigned to S03.
- Private artifact app-route behavior; assigned to S04.
- Full matrix/all-seam closure; assigned to S05.
- Browser/frontend flows, real PHI, live providers, production exploitation, CDN/object-storage behavior.

## Notes for Tester

This UAT is evidence-driven. Trust the committed summary/evidence artifacts only when they were generated by a passing root runner invocation with teardown complete and redaction validated.

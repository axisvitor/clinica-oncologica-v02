## Summary

Depth: targeted/deep security closeout research. S06 is primarily a proof/documentation slice, but it has one live remediation risk: `.gsd/REQUIREMENTS.md` still lists **R007 active** because generated Taskiq report PDFs still include sanitized free-form `report_type` in artifact filenames and task log context. Sanitization is not redaction when `report_type` can contain patient names or other PHI. Do not validate the milestone until R007 is either fixed and retested, or explicitly kept active with a documented blocker.

- **R010 primary:** reusable two-doctor/two-patient negative authorization proof across messages, quiz, flow responses, flow overrides, reports and upload/report serving.

- **R012/R013/R014 deferred:** medium hardening, proof gaps, and production-like runtime harness must be explicitly carried forward, not silently closed.

Most F-01..F-11 code fixes already exist and have focused tests from S01-S05. S06 should assemble one fresh integrated pytest run plus an evidence matrix, not rely only on historical task summaries.

### R007 current seam and likely change

Current code:

- `backend-hormonia/app/tasks/helpers/reports_helpers.py`:
  - `_sanitize_report_type(report_type)` strips path chars but keeps arbitrary alphanumeric/free-form content.
  - `_build_safe_report_path(base_dir, report_id, report_type)` builds `safe_filename = f"{report_id}_{_sanitize_report_type(report_type)}.pdf"`.
- `backend-hormonia/app/tasks/reports_taskiq.py`:
  - `safe_report_type = _sanitize_report_type(report_type)`.
  - `log_task_start`, invalid-patient warning, `log_task_error`, and `log_task_success` include `report_type=safe_report_type`.
  - Task result returns full `output_path` string; tests currently only assert patient UUID/full output path not in logs.
- `backend-hormonia/tests/tasks/test_reports_tasks.py` currently expects `output_path.name == f"{report_id}_medicalpatient-name.pdf"` for `report_type="medical/../../patient-name"`, which demonstrates the active R007 gap.

Recommended closure: either use report ID only (`{report_id}.pdf`) or a strict non-PHI allowlist such as `{report_id}_{category}.pdf` where unknown/free-form values map to `medical`/`other`. Also remove free-form `report_type` from log extras or log only the allowlisted category. Add tests that prove sentinel values like `patient-name`, `jane-doe`, tokens, and path segments do not appear in filename, task result path, or structured log records.

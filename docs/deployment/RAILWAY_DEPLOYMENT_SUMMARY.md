## 🔑 Environment Variables Strategy

### Critical Variables (Generate Securely)

```bash
# Generate SECRET_KEY
python -c "import secrets; print(secrets.token_urlsafe(32))"

# Generate JWT_SECRET_KEY
python -c "import secrets; print(secrets.token_urlsafe(32))"

# Generate ENCRYPTION_KEY
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

### Backend Variables (Required)
- `DATABASE_URL` - Auto-generated by Railway PostgreSQL
- `REDIS_URL` - Auto-generated by Railway Redis
- `SECRET_KEY` - Application secret (min 32 chars)
- `JWT_SECRET_KEY` - JWT signing key (min 32 chars)
- `ENCRYPTION_KEY` - Fernet encryption key
- `VITE_SUPABASE_URL` - Supabase project URL
- `VITE_SUPABASE_ANON_KEY` - Supabase anonymous key
- `GEMINI_API_KEY` - Google Gemini AI API key
- `WHATSAPP_API_URL` - Evolution API URL
- `WHATSAPP_API_KEY` - Evolution API key
- `WHATSAPP_INSTANCE_NAME` - WhatsApp instance name

### Frontend Variables (Required)
- `VITE_API_URL` - Backend API endpoint (e.g., `https://backend-api.railway.app/api/v1`)
- `VITE_API_BASE_URL` - Backend base URL
- `VITE_WS_BASE_URL` - WebSocket URL
- `VITE_SUPABASE_URL` - Same as backend
- `VITE_SUPABASE_ANON_KEY` - Same as backend
- `NODE_ENV=production`

## 📈 Scaling

### Horizontal Scaling
```bash
# Backend API (can have multiple replicas)
railway service scale --replicas 3 backend-api

# Frontend (can have multiple replicas)
railway service scale --replicas 2 frontend

# Celery Worker (can have multiple replicas)
railway service scale --replicas 4 celery-worker

# Celery Beat (MUST have only 1 replica - scheduler)
# Do not scale Beat service!
```

### Vertical Scaling
```bash
# Increase memory/CPU
railway service scale --memory 2048 backend-api
railway service scale --cpu 2 backend-api
```

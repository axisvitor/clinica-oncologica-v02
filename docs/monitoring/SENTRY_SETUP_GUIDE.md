# Sentry Monitoring Setup Guide\n\nComprehensive guide for setting up Sentry monitoring across all three systems of Clínica Oncológica.\n\n## Overview\n\nThis implementation provides comprehensive error tracking, performance monitoring, and custom context for:\n\n1. **Backend (FastAPI)** - Error tracking, performance monitoring, database operations\n2. **Frontend (React/Vite)** - Error boundaries, session replay, Web Vitals\n3. **Quiz Interface (Next.js)** - Quiz-specific tracking, API monitoring, client/server errors\n\n## Quick Start\n\n### 1. Create Sentry Projects\n\n1. Sign up at [sentry.io](https://sentry.io)\n2. Create three projects:\n   - `clinica-oncologica-backend` (Python/FastAPI)\n   - `clinica-oncologica-frontend` (React)\n   - `clinica-oncologica-quiz` (Next.js)\n3. Get DSN from each project's settings\n\n### 2. Environment Configuration\n\nCopy the example environment files and fill in your values:\n\n```bash\n# Backend\ncp backend-hormonia/.env.sentry.example backend-hormonia/.env\n\n# Frontend  \ncp frontend-hormonia/.env.sentry.example frontend-hormonia/.env\n\n# Quiz Interface\ncp quiz-mensal-interface/.env.sentry.example quiz-mensal-interface/.env.local\n```\n\n### 3. Install Dependencies\n\n```bash\n# Backend\ncd backend-hormonia\npip install sentry-sdk[fastapi,sqlalchemy,redis,httpx]\n\n# Frontend\ncd frontend-hormonia\nnpm install @sentry/react @sentry/tracing @sentry/integrations @sentry/replay\n\n# Quiz Interface\ncd quiz-mensal-interface\nnpm install @sentry/nextjs @sentry/tracing @sentry/integrations @sentry/replay\n```\n\n## Backend Integration (FastAPI)\n\n### Initialize Sentry\n\n```python\n# main.py\nfrom app.monitoring.sentry_config import SentryConfig, SentryMiddleware\n\n# Initialize Sentry\nSentryConfig.init_sentry()\n\n# Add middleware\napp.add_middleware(SentryMiddleware)\n```\n\n### Usage Examples\n\n```python\nfrom app.monitoring.sentry_config import (\n    SentryConfig, \n    monitor_performance,\n    capture_api_error,\n    monitor_quiz_session\n)\n\n# Set user context\nSentryConfig.set_user_context(\n    user_id=\"user123\",\n    email=\"user@example.com\",\n    role=\"patient\"\n)\n\n# Monitor Configuration\n\nRecommended alerts in Sentry:\n\n1. **Error Rate > 5%** in last 5 minutes\n2. **Response Time > 2s** for API endpoints\n3. **Quiz Completion Rate < 80%**\n4. **Database Query Time > 1s**\n5. **JavaScript Errors** affecting >10 users\n\n## Sampling Recommendations\n\n### Development\n- Traces: 100% (`1.0`)\n- Session Replay: 100% (`1.0`)\n- Profiles: 100% (`1.0`)\n\n### Staging\n- Traces: 20% (`0.2`)\n- Session Replay: 50% (`0.5`)\n- Profiles: 20% (`0.2`)\n\n### Production\n- Traces: 5-10% (`0.05-0.1`)\n- Session Replay: 10% (`0.1`)\n- Profiles: 5% (`0.05`)\n\n## Privacy Considerations\n\n### Data Scrubbing\n\n```python\n# Backend - Automatic PII scrubbing\nSENTRY_SEND_DEFAULT_PII=false\n\n# Custom data scrubbing\ndef before_send(event, hint):\n    # Remove sensitive fields\n    if 'request' in event:\n        headers = event['request'].get('headers', {})\n        headers.pop('authorization', None)\n        headers.pop('cookie', None)\n    return event\n```\n\n```typescript\n// Frontend - Mask sensitive elements\nnew Replay({\n  mask: ['.patient-ssn', '.medical-record-number', '.payment-info']\n})\n```\n\n## Troubleshooting\n\n### Common

## 🎓 LIÇÕES APRENDIDAS

### Técnicas

1. **Shell é suficiente para análise básica**
   - File system patterns (find, wc, grep) funcionam bem
   - Python AST necessário apenas para análise profunda

2. **Análise quantitativa revela problemas ocultos**
   - Flow services = 19% do código total (!!)
   - Cache vazio (0 LOC) ainda existe no projeto
   - "Enhanced" versions são red flags

3. **Padrões de nome indicam duplicação**
   - `ai*.py`, `cache*.py`, `flow*.py` revelam grupos óbvios
   - Múltiplos `*_service.py` para mesma coisa = problema

### Estratégicas

1. **Priorização por risco/impacto funciona**
   - Low-risk first = quick wins + confiança
   - High-risk last = mais tempo para planejar
   - Fases claras = progresso visível

2. **Métricas são essenciais**
   - Números concretos convencem stakeholders
   - LOC e arquivos são métricas simples mas poderosas
   - Visualização de redução (72%) é impactante

3. **Documentação antecipada poupa tempo**
   - Relatório gerado = baseline para consolidação
   - Roadmap claro = menos debates depois
   - Recomendações específicas = menos decisões ad-hoc
